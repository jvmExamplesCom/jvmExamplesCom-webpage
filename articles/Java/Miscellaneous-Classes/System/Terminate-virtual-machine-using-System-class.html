Terminating Virtual Machines using System Class in Java

Java provides a System class which holds useful methods and variables. One of these methods, called exit() , can be used to terminate the current Java virtual machine (JVM). This article will explain how to use this method and provide example usage.

The System Class

The System class is part of the java.lang package and provides several useful methods. The existence of this class was introduced in Java 1.0 and has been stable until the current version. It contains the following methods: 

          
           
public static void exit(int status): Terminates the currently running JVM. 
public static long currentTimeMillis(): Returns the current time in milliseconds since Jan 1, 1970.
public static void gc(): Runs the garbage collector.
As you can see, the set of methods provided by the System class can be used in many different ways. The exit() method is the only one that can be used to terminate the JVM.

Using the exit() Method

The exit() method takes an integer argument which is the status code and is returned to the operating system when the program finishes. The status code can be any integer value from 0 to 255. The exit() method throws a SecurityException if a security manager exists and its checkExit() method does not allow termination of the current JVM. The following example shows how to use the exit() method correctly: 

          
           
public class Main { 
    public static void main(String[] args) { 
        int status = 0; 
        try { 
            // your code here 
        } catch (Exception e) { 
            status = 1; 
        } finally { 
            System.exit(status); 
        } 
    } 
}
In the above example, the status variable is initialized to zero and then set to one when an exception is thrown. The System.exit() command is then called with the status variable as its argument.

Conclusion

The System class in Java provides several useful methods, including the exit() method which can be used to terminate the current JVM. It takes an integer argument which is the status code returned to the operating system when the program finishes.
