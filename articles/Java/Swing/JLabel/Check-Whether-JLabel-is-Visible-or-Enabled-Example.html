Check Whether JLabel is Visible or Enabled Example

The Java Swing JLabel class provides a constructor which allows the developer to specify whether the JLabel should be initially visible or enabled. Being able to check the visibility and enablement of a JLabel can be useful for a variety of reasons, such as debugging or creating logic to control when content appears in the application.

Example Code Demonstrating How to Check Whether a JLabel is Visible or Enabled

In this example code snippet, we will show how to check whether a given JLabel is currently visible or enabled. First, we must create an instance of the JLabel class:


          
           
JLabel label = new JLabel("My Label");

Next, we use the methods isVisible() and isEnabled() to check the state of the JLabel:


          
           
boolean isEnabled = label.isEnabled();
boolean isVisible = label.isVisible();

We can then use the returned boolean values from these methods to determine whether the JLabel should be visible or enabled. For example, if we want the JLabel to be visible, but not enabled, we can set the respective properties accordingly:


          
           
label.setEnabled(false);
label.setVisible(true);

If the desired conditions are already satisfied, the developer can use the returned boolean values to trigger some custom logic:


          
           
if (isEnabled &amp;&amp; isVisible) {
  // Run some additional logic
}

Finally, the developer can check whether the settings have been applied correctly by outputting the current state of the JLabel:


          
           
System.out.println("Enabled: " + label.isEnabled());
System.out.println("Visible: " + label.isVisible());

After running the above code example, the output will look like this:


          
           
Enabled: false
Visible: true
