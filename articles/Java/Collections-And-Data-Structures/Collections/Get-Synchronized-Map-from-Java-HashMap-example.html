Synchronized Map from Java HashMap Example

Java provides a set of core collections that implement the Map interface to store and manipulate data. HashMap is one of the most commonly used Map implementations, which allows null key and values. In a multithreaded environment, modifications should be synchronized to ensure data integrity. This article shows how to get a synchronized map (java.util.Collections.synchronizedMap(Map)) from the original HashMap by applying an example.

Creating a HashMap Instance

To demonstrate how to get a synchronized map, let's create a HashMap instance firstly. 

          
           
HashMap<String, String> hashmap = new HashMap<String, String>();
hashmap.put("apple", "red");
hashmap.put("orange", "orange");
hashmap.put("grape", "purple");
hashmap.put("strawberry", "pink");
We can print out the keys and values of the hashmap instance as follows: 

          
           
for(String key : hashmap.keySet()) {
   System.out.println(key + "\t" + hashmap.get(key));
}
The output of the code will be the following: 

          
           
apple	red
orange	orange
grape	purple
strawberry	pink

Getting Synchronized Map

We are able to create a synchronized map from this HashMap instance by using the 

          
           
Collections.synchronizedMap()
method as follows: 

          
           
Map<String, String> synchronizedMap = Collections.synchronizedMap(hashmap);
The returned synchronizedMap is a synchronized wrapper of the original map. Any modifications to the returned map will be synchronized on the original map.

Example of Using Synchronized Map

Let's come up with an example to demonstrate how to use the synchronizedMap. The thread class MyRunnable is defined as follows: 

          
           
class MyRunnable implements Runnable {
    private Map<String, String> map;

    public MyRunnable(Map<String, String> map) {
        this.map = map;
    }

    @Override
    public void run() {
        for (Map.Entry<String, String> entry : map.entrySet()) {
            System.out.println(Thread.currentThread().getName() 
                    + ":\t" + entry.getKey() + "\t" + entry.getValue());
        }
    }
}
It prints out all the keys and values in the map instance. The main method looks like this: 

          
           
public static void main(String[] args) {
    Map<String, String> synchronizedMap = Collections.synchronizedMap(hashmap);
    MyRunnable myRunnable = new MyRunnable(synchronizedMap);

    Thread thread1 = new Thread(myRunnable);
    Thread thread2 = new Thread(myRunnable);

    thread1.start();
    thread2.start();
}
Two threads are created to run the same MyRunnable. Meanwhile, any modifications to the synchronizedMap will be synchronized on the original HashMap. The output of the code might be: 

          
           
Thread-0:	apple	red
Thread-0:	orange	orange
Thread-0:	grape	purple
Thread-1:	strawberry	pink
Thread-1:	apple	red
Thread-1:	orange	orange
Thread-1:	grape	purple
As the output shown, threads accessing the same map instance will get the same mapping results at any given time.

Conclusion

This article has demonstrated how to get a synchronized map from a HashMap instance using the java.util.Collections.synchronizedMap() method. A simple example is also provided to show how to use it in a multithreaded environment.
