How to Create a HashMap in Java

The HashMap class is part of the Java Collection Framework and it is an advanced version of the Map interface. It is used to store key-value pairs and is based on the idea of hashing. It offers constant time performance for basic operations like get() and put(). In this tutorial, we will be looking at the various constructors that are available in HashMap. We will be looking at how they work and also examining a few examples of how they can be used.

HashMap Constructors

The HashMap class has four different constructors that we can use. The first one is the no-args constructor which creates an empty HashMap with the default initial capacity of 16 and load factor of 0.75. This constructor is used when we want to create a HashMap and add the key-value pairs later. Here's an example of how to use it: 

          
           
//No-args constructor
HashMap<String, Integer> hmap = new HashMap<>;
The second constructor takes a single argument which is a Map object. This constructor creates a new HashMap with the same mappings as the specified map. Here's an example of how to use it: 

          
           
//Constructor with a map
Map<String, Integer> map = new HashMap<String, Integer>();

map.put("one", 1);
map.put("two", 2);

HashMap<String, Integer> hmap = new HashMap<String, Integer>(map);
The third constructor takes two arguments – initial capacity and load factor. This constructor creates an empty HashMap with a capacity specified by the initial capacity argument and a load factor specified by the load factor argument. Here's an example of how to use it: 

          
           
//Constructor with initial capacity and load factor
HashMap<String, Integer> hmap = new HashMap<String, Integer>(20, 0.75f);
The fourth constructor also takes two arguments – initial capacity and load factor. However, this constructor also accepts a Boolean argument which specifies whether or not the map allows null keys. This constructor creates an empty HashMap with a capacity specified by the initial capacity argument and a load factor specified by the load factor argument, and allows or disallows null keys depending on the boolean value given. //Constructor with initial capacity, load factor and allowNulls parameter
