How to Iterate LinkedHashMap in Reverse Order

LinkedHashMap is an implementation of the Map interface that maintains insertion order. The insertion order is preserved by maintaining a doubly-linked list running through all of its entries. This makes LinkedHashMap ideal for iterating over in its insertion order, or any order imposed by a comparator.

Reverse Iteration with LinkedHashMap

Iterating over a LinkedHashMap in reverse order usually requires the use of a custom iterator. To reverse a LinkedHashMap, we can use a “descending” iterator, which will iterate through the LinkedHashMap in reverse order. The following example shows how to create a descending iterator for a LinkedHashMap: 

          
           
Iterator<Map.Entry<Integer, String>> it = linkedHashMap.entrySet().iterator();
Iterator<Map.Entry<Integer, String>> descendingIt = new Iterator<Map.Entry<Integer, String>>() {
    private Iterator<Map.Entry<Integer, String>> forwardIt = it;

    @Override
    public boolean hasNext() {
        return forwardIt.hasPrevious();
    }

    @Override
    public Map.Entry<Integer, String> next() {
        return forwardIt.previous();
    }
};
Now that we have our descending iterator, we can iterate over the LinkedHashMap in reverse order by looping over it: 

          
           
while (descendingIt.hasNext()) {
    Map.Entry<Integer, String> entry = descendingIt.next();
    System.out.println(entry.getKey() + " : " + entry.getValue());
}
