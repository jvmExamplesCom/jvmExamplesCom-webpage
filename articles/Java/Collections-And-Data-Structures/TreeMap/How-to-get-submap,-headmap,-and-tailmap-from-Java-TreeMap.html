Getting Submap, Headmap, and Tailmap from Java TreeMap

TreeMap is a powerful class in the Java API which allows for an efficient way to store and retrieve key-value pairs of data. Its usefulness comes from its ability to perform very fast lookups based on keys. Additionally, it provides multiple ways to get submaps, headmaps, and tailmaps. This article will discuss how to get these collections from TreeMap.

Submap

A submap is a collection created from a portion of the original TreeMap. It takes two parameters, a starting key and an ending key, to create a view of the Map between these two keys. This view is a shallow copy, meaning that any changes made to the original map will affect the submap, and vice versa.

For example, let’s say you wanted to create a submap with keys between “cat” and “dog” in our TreeMap. You could do so by using the following code:


          
           
TreeMap<String, String> map = new TreeMap<>();
map.put("ant", "insect");
map.put("bat", "mammal");
map.put("cat", "mammal");
map.put("dog", "mammal");
map.put("elephant", "mammal");

NavigableMap<String, String> submap = map.subMap("cat", "dog");

The resulting submap would contain all entries between “cat” and “dog”, including both the keys and their corresponding values:


          
           
{cat=mammal, dog=mammal}

Headmap

A headmap is a collection created from the first part of a TreeMap which contains keys less than or equal to a given key. This view is also a shallow copy, so changes made to the original map will be reflected in the headmap.

For example, let’s say you wanted to create a headmap containing all keys up to “dog” in our TreeMap. You could do this by using the following code:


          
           
NavigableMap<String, String> headmap = map.headMap("dog", true);

The resulting headmap would contain the following key-value pairs:


          
           
{ant=insect, bat=mammal, cat=mammal, dog=mammal}

Tailmap

A tailmap is created from the second part of a TreeMap which contains keys greater than or equal to a given key. Just like the submap and headmap, this view is a shallow copy and will reflect any changes made to the original map.

For example, let’s say you wanted to create a tailmap containing all keys beginning with “dog” in our TreeMap. You could do this by using the following code:


          
           
NavigableMap<String, String> tailmap = map.tailMap("dog", true);

The resulting tailmap would contain the following key-value pairs:


          
           
{dog=mammal, elephant=mammal}

These are just some of the ways to use the powerful TreeMap class to get submaps, headmaps, and tailmaps. These views provide an efficient way to access portions of the map without needing to duplicate the data. Knowing how to use these can come in handy when dealing with large datasets.
