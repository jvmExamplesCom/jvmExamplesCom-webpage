Comparing Two HashSet Objects Using The Equals Method

The 

          
           
java.util.HashSet<E>
class is an implementation of the 
Set
interface that uses a 

          
           
HashMap<E,Object>
to store its elements. This makes it much faster and more efficient when compared to other implementations of the Set interface, such as 
TreeSet
. One important thing to be aware of with HashSet is that the order in which elements are added to the set can influence the order in which they are returned when iterating over the Set.

When working with HashSet, it can often be useful to compare two instances of the class for equality. Fortunately, the equals method is provided for this purpose in the Java API. This article will demonstrate how to use the equals method to compare two HashSet objects.

Using The Equals Method To Compare Two HashSet Objects

The equals method can be used to compare two different HashSet instances for equality. It takes as input another Object and returns a boolean value indicating whether or not the two objects are equal. The following code example demonstrates how to use the equals method to compare two HashSet objects:


          
           
HashSet<String> set1 = new HashSet<>();
set1.add("Apple");
set1.add("Orange");
set1.add("Banana");

HashSet<String> set2 = new HashSet<>();
set2.add("Apple");
set2.add("Orange");
set2.add("Banana");

if (set1.equals(set2)) {
    System.out.println("The two sets are equal!");
}
else {
    System.out.println("The two sets are not equal!");
}

In this example, we create two HashSet objects and add three elements each to them. We then use the equals method to compare the two sets and print the appropriate output. If we execute this code, it will print 


          
           
The two sets are equal!
as expected.

Comparing Sets With Different Ordering

As mentioned previously, the order in which elements are added to a HashSet instance can influence the order in which they are returned when iterating over the set. This means that it is possible to have two sets that contain the same elements, but the ordering of the elements could be different. When using the equals method to compare these two sets, the result will be false, indicating that the sets are not equal. To illustrate this behavior, consider the following example:


          
           
HashSet<String> set1 = new HashSet<>();
set1.add("Apple");
set1.add("Orange");
set1.add("Banana");

HashSet<String> set2 = new HashSet<>();
set2.add("Banana");
set2.add("Orange");
set2.add("Apple");

if (set1.equals(set2)) {
    System.out.println("The two sets are equal!");
}
else {
    System.out.println("The two sets are not equal!");
}
This code produces the output 

          
           
The two sets are not equal!
, even though the two sets contain the same elements. This is because the order in which the elements are added to the sets is different, which results in the two sets being considered unequal.

Conclusion

In this article, we've seen how to use the equals method to compare two HashSet objects in Java. We've also seen that when comparing sets with different ordering, the result can be false even if the two sets contain the same elements.
