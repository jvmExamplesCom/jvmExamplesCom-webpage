org.eclipse.jetty.server.handler.ContextHandlerCollection

The org.eclipse.jetty.server.handler.ContextHandlerCollection is an handler collection which maintains a list of ContextHandler instances. It allows the configuration of multiple web applications within one Jetty server instance. ContextHandlerCollection can be used to manage multiple contexts and resources, while still allowing them to be accessible from the same URL.

Example Usage

ContextHandlerCollection can be used in many different scenarios. Here is an example of how it can be used to configure multiple web applications:


          
           
ContextHandlerCollection contexts = new ContextHandlerCollection();

// Create and configure a context for the app1 web application
ContextHandler context1 = new ContextHandler("/app1");
context1.setResourceBase("/var/www/app1/");
context1.setHandler(new WebAppContext());

// Create and configure a context for the app2 web application
ContextHandler context2 = new ContextHandler("/app2");
context2.setResourceBase("/var/www/app2/");
context2.setHandler(new WebAppContext());

// Add the contexts to the collection
contexts.addHandler(context1);
contexts.addHandler(context2);

// Set the handler collection on the server
server.setHandler(contexts);

In this example, two web applications have been configured, each with its own set of resources and contexts. They can now both be accessed by the same server, using the same URL.
