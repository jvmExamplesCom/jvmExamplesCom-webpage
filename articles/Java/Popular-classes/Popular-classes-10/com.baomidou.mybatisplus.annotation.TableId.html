TableId Annotation in MyBatisPlus

MyBatisPlus is an open-source ORM framework that provides a dynamic object-oriented approach to querying databases and manipulating data. One of its many features is the 

          
           
@TableId
annotation, which is used to identify and associate fields in a database table with their associated Java objects. This annotation is essential for creating and maintaining relationships between databases and Java classes. In this article, we'll discuss the specifics of the

What Does the TableId Annotation Do?

The 

          
           
@TableId
annotation is used to declare an entity class field as primary key. It can be used on fields of type Integer, Long, String, or BigInteger. The annotation supports a few properties, such as “name”, “type”, and “generatedValue”.
  • name: This is the name of the primary key. It should correspond to the actual column name in the database table.
  • type: This is the type of the primary key. It can either be AUTO_INCREMENT, SEQUENCE, or NONE.
  • generatedValue: This property is used to define the strategy for generating primary keys. It must be set if the type of the primary key is AUTO_INCREMENT or SEQUENCE.
