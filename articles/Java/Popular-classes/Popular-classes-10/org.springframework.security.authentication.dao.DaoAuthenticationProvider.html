org.springframework.security.authentication.dao.DaoAuthenticationProvider

The DaoAuthenticationProvider class is a concrete implementation of AuthenticationProvider for authenticating users using user details provided by a UserDetailsService. It handles authentication in the same way as basic authentication but stores the credentials in an encrypted format.

Usage Example

The following example shows how to use the DaoAuthenticationProvider to authenticate a user.


          
           
@Autowired
private UserDetailsService userDetailsService;

@Bean
public DaoAuthenticationProvider authProvider() {
    DaoAuthenticationProvider authProvider = new DaoAuthenticationProvider();
    authProvider.setUserDetailsService(userDetailsService);
    authProvider.setPasswordEncoder(passwordEncoder());
    return authProvider;
}

Authentication Process

When authenticating, the DaoAuthenticationProvider will use the PasswordEncoder it was configured with to compare the credentials supplied by the client to the credentials stored.

If the credentials match, the authentication process is successful and the authenticated user is returned.

Username Password Authentication

The DaoAuthenticationProvider can be configured to perform username/password authentication by providing a UserDetailsService that can look up the credentials for the supplied username. The following example shows how to set up a UserDetailsService:


          
           
@Bean
public UserDetailsService userDetailsService() {
    return new InMemoryUserDetailsManager(
        User.withDefaultPasswordEncoder()
            .username("user")
            .password("password")
            .roles("USER")
            .build()
    );
}

When authenticating with a username and password, the supplied credentials will be compared against the credentials stored in the UserDetailsService.

Customizing Password Encoding

The DaoAuthenticationProvider can be configured to use a custom PasswordEncoder to encode passwords before they are compared to the credentials stored in the UserDetailsService. The following example shows how to configure the DaoAuthenticationProvider with a BCryptPasswordEncoder:


          
           
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}

@Bean
public DaoAuthenticationProvider authProvider() {
    DaoAuthenticationProvider authProvider = new DaoAuthenticationProvider();
    authProvider.setUserDetailsService(userDetailsService());
    authProvider.setPasswordEncoder(passwordEncoder());
    return authProvider;
}

The DaoAuthenticationProvider will now use the BCryptPasswordEncoder to encode the passwords before comparing them with the credentials stored in the UserDetailsService.

The DaoAuthenticationProvider is a useful AuthenticationProvider for authenticating users using credentials stored in a UserDetailsService. It can be configured to use a custom PasswordEncoder to encode passwords before they are compared to the credentials stored in the UserDetailsService.
