Creating and Configuring an ExoPlayer2 Player

Creating an instance of the ExoPlayer2 player is accomplished with the following code: 

          
           
Player player = ExoPlayerFactory.newSimpleInstance(context);
The Player instance returned is configured with default parameters and is ready to use. You can also configure the Player by passing in a Player.Parameters object like this: 

          
           
Player.Parameters params = new Player.Parameters();
params.setHls(true);
Player player = ExoPlayerFactory.newSimpleInstance(context, params);
In this example, we have set the Hls parameter to true, which means that the Player will prefer to use the HLS streaming protocol for media playback if it is available.

Playing Audio and Video

Once you have created and configured your ExoPlayer2 Player instance, you can begin playing audio or video content. Playing audio files is straightforward, simply call the Player.prepare() method and pass in a Uri that points to the audio file. For example, to play an MP3 file located at http://example.com/myfile.mp3, use the following code: 

          
           
Uri uri = Uri.parse("http://example.com/myfile.mp3");
player.prepare(uri);
player.setPlayWhenReady(true);
Playing video files is slightly different, as you must also specify a VideoRenderer in order to render the video frames. You can use the ExoPlayer2 built-in renderer, like this: 

          
           
Uri uri = Uri.parse("http://example.com/myfile.mp4");
VideoRenderer renderer = ExoPlayerFactory.newFromType(VideoRenderer.TYPE_DEFAULT, context);
player.prepare(uri, renderer);
player.setPlayWhenReady(true);

Subtitles and Captions

ExoPlayer2 also supports subtitles and captions, with support for a variety of text formats such as SRT, WebVTT, and others. To enable subtitles, you must specify a TextRenderer when preparing the Player: 

          
           
Uri uri = Uri.parse("http://example.com/myfile.mp4");
VideoRenderer videoRenderer = ExoPlayerFactory.newFromType(VideoRenderer.TYPE_DEFAULT, context);

You can also specify a Uri pointing to a subtitle file to be parsed and displayed along with the video file.
