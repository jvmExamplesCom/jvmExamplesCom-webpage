Introduction to javax.jms.DeliveryMode

The 

          
           
javax.jms.DeliveryMode
class is an enumeration type used to indicate the strength or durability of a message delivery mechanism. It defines two message delivery modes, Persistent and Non-Persistent.

Persistent DeliveryMode

The Persistent DeliveryMode is used when we want the message delivery to be stronger or more reliable than the default delivery mode. It ensures that the message is delivered even in cases of system failure or crash. This is done by storing the message until it is successfully delivered to the target destination. 

          
           
//Creating a JMS connection
Connection connection = ConnectionFactory.createConnection();

//Creating a queue
String queueName = "myQueue";
Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
Queue queue = session.createQueue(queueName);

//Create a producer for sending messages
MessageProducer producer = session.createProducer(queue);

// Set the delivery mode to persistent
producer.setDeliveryMode(DeliveryMode.PERSISTENT);

Non-Persistent DeliveryMode

The Non-Persistent DeliveryMode is used when the message delivery time is not as important as the reliability of the delivery. The message is delivered at the earliest convenience of the messaging system, but it is not stored in case of a system failure. 

          
           
//Creating a JMS connection
Connection connection = ConnectionFactory.createConnection();

//Creating a queue
String queueName = "myQueue";
Session session = connection.createSession(false, Session.AUTO_ACKNOWLEDGE);
Queue queue = session.createQueue(queueName);

//Create a producer for sending messages
MessageProducer producer = session.createProducer(queue);

//Set the delivery mode to non-persistent
producer.setDeliveryMode(DeliveryMode.NON_PERSISTENT);

Conclusion

The javax.jms.DeliveryMode class provides a mechanism to define the strength and durability of message delivery. With this class, we can choose between two delivery modes, Persistent and Non-Persistent. Depending on our requirements, one or both of these modes can be used to ensure that the messages are delivered reliably and in a timely manner.
