Overview of android.arch.persistence.room.PrimaryKey

android.arch.persistence.room.PrimaryKey is an annotation that is used to designate a field as primary key in a Room database table. This annotation allows developers to mark a field, or collection of fields, as the primary key for a Room database table. It also allows them to customize the order of the fields that make up the key.

Using the PrimaryKey Annotation

The PrimaryKey annotation can be used by itself on a single field to designate it as the table's primary key. For example: 

          
           
@PrimaryKey
int id;
If you want to create a composite key with multiple fields, use the PrimaryKey annotation on each field and set the order property to determine their order in the composite key: 

          
           
@PrimaryKey(order = 0) 
int id;

@PrimaryKey(order = 1)
String name;
The order property is important because it determines the order of the fields in the composite key. The first field in the primary key will have order = 0, the second field will have order = 1, and so on.

Adding Auto Generated Primary Key

When Room generates a value for this field, it will be a unique identifier that is generated using the device's internal clock. This is useful for creating unique records that will never conflict with one another.

Adding an Already Existing Primary Key

If you are migrating data from an existing table, you may want to keep the primary key from the original table. In this case, you can use the @ExistingPrimaryKey annotation to indicate that the primary key is already set and should not be changed. For example:
