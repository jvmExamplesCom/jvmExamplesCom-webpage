What is org.junit.jupiter.api.Tag?

org.junit.jupiter.api.Tag is a JUnit 5 annotation used as an alternative to the standard @Test annotation. It allows developers to tag their JUnit tests with arbitrary keywords which can then be used for filtering purposes when running the tests.

How to Use org.junit.jupiter.api.Tag?

The Tag annotation can be used in two different ways:

  1. On the method level, which applies the tag to all test cases within the method:

          
           
@Tag("MyTag")
@Test
public void myTest() {
   // Test logic here
}
  1. On the class level, which applies the tag to all test cases within the class:

          
           
@Tag("MyTag")
class MyTestClass {
   @Test
   public void myTest() {
      // Test logic here
   }
}

In both cases, a single tag or multiple tags can be specified.

Examples of org.junit.jupiter.api.Tag

It is possible to use the Tag annotation for various purposes, such as segregating tests based on type, environment, platform, feature, etc. Below are some examples of how to use the Tag annotation:

  • Grouping tests by environment: 
    
            
             
@Tag("production")
@Test
public void productionTest() {
   // Production environment test logic here
}

@Tag("staging")
@Test
public void stagingTest() {
   // Staging environment test logic here
}
  • Grouping tests by type: 
    
            
             
@Tag("integration")
@Test
public void integrationTest() {
   // Integration test logic here
}

@Tag("performance")
@Test
public void performanceTest() {
   // Performance test logic here
}
  • Grouping tests by feature: 
    
            
             
@Tag("login")
@Test
public void logInTest() {
   // Login test logic here
}

@Tag("account")
@Test
public void accountTest() {
   // Account test logic here
}

Conclusion

The org.junit.jupiter.api.Tag annotation is a powerful tool that can be used to organize and filter JUnit tests. By grouping tests into logical categories, it is much easier to run only certain tests that are relevant to the current development task, thereby saving time and resources in the long run.
