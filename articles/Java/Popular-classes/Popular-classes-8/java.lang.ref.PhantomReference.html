Java.lang.ref.PhantomReference

The java.lang.ref.PhantomReference class is an abstract class in the Java language which allows developers to create objects which are less susceptible to garbage collection. This class enables developers to retain references to objects without having the objects stay "alive" in memory, like other types of references do.

When an object is no longer strongly-reachable, it is marked for garbage collection. When this happens, the garbage collector can call two methods in the PhantomReference class: get() and clear(). Get returns null, and clear removes the reference from the garbage collector's reference queue.

A phantom reference is useful when you need to refer to an object that could be garbage collected, but still be able to refer to it. It is a special type of reference that is not cleared by the garbage collector by default. A phantom reference is never used to directly refer to the object, but rather to track when the object has been garbage collected. Once the referent is garbage collected, then the PhantomReference object is added to the ReferenceQueue.
