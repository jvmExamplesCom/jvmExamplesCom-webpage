org.springframework.data.jpa.domain.support.AuditingEntityListener

AuditingEntityListener is a convenient base class for providing auditing information for entities in Spring Data applications. It provides a basic set of operations for persisting creation and modification information for entities with an @Audited annotation.

Introduction

AuditingEntityListener allows you to configure automatic auditing of entities in your Spring Data JPA Applications. It provides the following features:

  • Automatic capture of the username and timestamp of entity create and updates events.
  • Support for customizing the username field (e.g. changing it from "createdBy" to "creator").
  • Integration with Spring Security's Authentication object, allowing the username to be captured based on the current logged in user.

Class Definition


          
           
@Configuration
@EnableJpaAuditing
public class AuditingEntityListener extends AbstractAuditingEntityListener {

  @Override
  public void configure(JpaProperties jpa) {
     jpa.setAuditingEnabled(true);
  }

}

The above code declares the AuditingEntityListener class with the EnableJpaAuditing annotation which enables automatic auditing in the application context. The configure() method is then overridden from AbstractAuditingEntityListener and sets the jpa property AuditingEnabled to true.

Annotating entities with @Audited

Once the AuditingEntityListener has been declared, all entities that need to be audited must be annotated with the @Audited annotation. This will trigger the listener when an entity is created or updated and capture the information of who created or updated the entity as well as the timestamp.


          
           
@Entity
@Table(name = "users")
@Audited
public class User {

  @Id
  @GeneratedValue(strategy = GenerationType.IDENTITY)
  private Long id;

  private String name;
  
  // Constructors, getters and setters omitted for brevity

}

Configuring AuditingEntityListener

The AuditingEntityListener can be configured to provide additional information such as the user responsible for persisting entities, the date format and the handler for determining the currently logged in user. To configure the listener, an AuditingEntityListenerRegistry can be used.


          
           
AuditingEntityListenerRegistry registry = new AuditingEntityListenerRegistry();

registry.register(User.class, new AuditingEntityListener());

registry.addUsernameField("createdBy");
registry.addTimestampField("createdAt");
registry.addUsernameField("updatedBy");
registry.addTimestampField("updatedAt");

AuditingEntityListener auditListener = 
  registry.getAuditingEntityListener(User.class);

auditListener.setUsernameHandler(new UsernameHandler() {

  @Override
  public String getCurrentUsername() {
     return SecurityContextHolder.getContext().getAuthentication().getName();
  }

});

In the above code, the AuditingEntityListenerRegistry is used to register the AuditingEntityListener for the User entity. The addUsernameField() and addTimestampField() methods are used to add custom username and timestamp fields to the entity. The username handler is also configured so that the currently logged in user's name can be retrieved from the SecurityContext when creating or updating entities.

Once the AuditingEntityListener is configured, its auditing operations will be enabled for all entities annotated with @Audited. When an entity is saved or updated, the listener will capture the username and timestamp of the operation and store it in the corresponding fields.
