Introduction

The 

          
           
org.eclipse.aether.RepositorySystem
facilitates the *resolution* of artifacts and their transitive dependencies from a set of repositories. It allows Java developers to easily and reliably use external libraries in their projects. The repository system is plugin-based, allowing custom implementations to be plugged in as needed.

Getting Started

Before you can use 

          
           
org.eclipse.aether.RepositorySystem
, you must first add a dependency to your project. Add the following dependency to your project's 

          
           
pom.xml
file: 

          
           
<dependency>
    <groupId>org.eclipse.aether</groupId>
    <artifactId>aether-api</artifactId>
    <version>1.1.0</version>
</dependency>
Once the dependency has been added, you are ready to start using 

          
           
org.eclipse.aether.RepositorySystem
in your project.

// create a new instance of the RepositorySystem

RepositorySystem

Resolving Artifacts with RepositorySystem

The most common use case for the 

          
           
org.eclipse.aether.RepositorySystem

// create an instance of the RepositorySystemSession
