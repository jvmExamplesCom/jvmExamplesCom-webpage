org.apache.ibatis.session.Configuration

The 

          
           
org.apache.ibatis.session.Configuration
class is the heart of MyBatis configuration. It is the primary interface for configuring MyBatis and a complete reference of available configuration options. It provides access to user-defined SQL maps, database transactions, and persistent objects. This article will discuss how to configure and use the Configuration class.

Configuration Basics

The 

          
           
Configuration
class provides several methods for configuring a MyBatis application. The most basic configuration consists of specifying a database connection and some SQL maps: 

          
           
Configuration config = new Configuration();
config.addDataSource(dataSource);
config.addMapping("my-sql-map.xml");
The 

          
           
Configuration
class can also be used to specify type handlers, interceptors, and other options. For example, to register a type handler with the Configuration: 

          
           
TypeHandler typeHandler = new MyTypeHandler();
config.getTypeHandlerRegistry().register(typeHandler);
To register an interceptor: 

          
           
Interceptor interceptor = new MyInterceptor();
config.addInterceptor(interceptor);

Configuration
object is setup, it is then used to create a 

          
           
SqlSessionFactory
. The 

          
           
SqlSessionFactory
is the primary interface for interacting with MyBatis. All database operations are done through the factory and all configuration options set in the 

          
           
Configuration

SqlSessionFactory factory = config.buildSqlSessionFactory();

SqlSessionFactory
can then be used to create 

          
           
SqlSession

SqlSession session = factory.openSession();

Using the Configuration

Configuration

Configuration

TypeHandlerRegistry registry = config.getTypeHandlerRegistry();
