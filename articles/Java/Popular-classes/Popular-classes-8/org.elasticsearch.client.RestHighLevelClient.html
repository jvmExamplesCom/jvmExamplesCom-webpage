Introduction to org.elasticsearch.client.RestHighLevelClient

The 

          
           
org.elasticsearch.client.RestHighLevelClient
is a library provided by the Elasticsearch Java Rest Client that can be used for communication between the client and an Elasticsearch cluster. It provides a high-level API with methods that make it easy to use the REST APIs of the Elasticsearch cluster. This library is built on top of the low-level REST client, 

          
           
org.elasticsearch.client.RestClient
, so users can either choose to use the high-level or low-level client to communicate with the cluster.

Using RestHighLevelClient

In order to use the RestHighLevelClient you must first create an instance of it. This can be done by providing the client with a set of nodes to connect to and credentials to authenticate with. 


          
           
RestHighLevelClient client = new RestHighLevelClient(
    RestClient.builder(
        new HttpHost("localhost", 9200, "http"),
        new HttpHost("localhost", 9201, "http")
    )
);

Once the client is created, you can call its methods to make requests to the Elasticsearch cluster. For example, the following code makes a search request to the cluster:


          
           
SearchRequest searchRequest = new SearchRequest();
SearchSourceBuilder searchSourceBuilder = new SearchSourceBuilder();
searchSourceBuilder.query(QueryBuilders.matchQuery("message", "Hello World"));
searchRequest.source(searchSourceBuilder);

SearchResponse searchResponse = client.search(searchRequest);
The search request returns a 
SearchResponse
object which contains the results of the search request. The response contains information such as the total number of hits, the took time (in milliseconds), and the list of documents that were matched by the query.

Configuring RestHighLevelClient

The RestHighLevelClient can also be configured to customize its behaviour. For example, you can configure the maximum number of retries when making requests to the cluster, the timeout for requests, and which headers to include in requests. 


          
           
client.setMaxRetryTimeoutMillis(6000);
client.setDefaultHeaders(Collections.singletonMap("MyHeader", "MyValue"));

Finally, the client can be closed when it is no longer needed. This will release any resources that have been allocated for the client and ensure that the connection to the cluster is closed. 


          
           
client.close();
