Introduction to Android.app.IntentService

Android.app.IntentService is a service class provided by the Android platform for performing asynchronous work. It is a subclass of android.app.Service and can be used to execute a long-running operation on a background thread. This class also helps to ensure that operations are properly handled when the system performs maintenance such as memory reclaiming.

Creating an IntentService

To create an IntentService, you must create a subclass of android.app.IntentService. The intentService class provides two methods that must be overridden in order to implement the service: onHandleIntent() and onCreate(). OnCreate() is called once when the service is first created, and onHandleIntent() is called each time the service receives an intent. 

          
           
public class MyIntentService extends IntentService {
  
    public MyIntentService() {
        super("MyIntentService");
    }
  
    @Override
    protected void onHandleIntent(Intent intent) {
        //Do your long running task here
    }

    @Override
    public void onCreate() {
        super.onCreate();
    }
}

Starting an IntentService

An IntentService can be started using an Intent object. The Intent object can contain any extra data needed to perform the requested operation, and it will be passed to the onHandleIntent() method. 

          
           
Intent intent = new Intent(this, MyIntentService.class);
intent.putExtra("param1", "value1");
startService(intent);

Stopping an IntentService

An IntentService can be stopped by calling stopService() on the Intent object used to start the service. 

          
           
stopService(intent);

Handling Results from an IntentService

The onHandleIntent() method of an intentService runs on a separate thread from the main thread. When the method completes, the results can be passed back to the main thread using a BroadcastReceiver. This allows the main thread to be updated with the results of the IntentService without blocking the main thread.
