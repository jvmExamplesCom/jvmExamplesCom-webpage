javax.persistence.GenerationType

Java Persistence API (JPA) is a Java specification for accessing and managing relational databases. It provides a set of standard object-relational mapping (ORM) APIs for defining, persisting, and querying entities and their associated objects. One of the most important elements of JPA is the GenerationType, which defines how an entityâ€™s id is generated.

Table Generation Strategies

The GenerationType enum provides four strategies for generating ids: TABLE, AUTO, SEQUENCE, and IDENTITY.

TABLE Strategy

The TABLE strategy is the simplest and least efficient of the strategies. A database table is used to store the generated values, and the next value is selected by performing a query. 

          
           
@Id
@GeneratedValue(strategy=GenerationType.TABLE)
private Long id;

AUTO Strategy

The AUTO strategy uses an internal database sequence or other internal mechanism for generating unique identifiers. It is far more efficient than the TABLE strategy, but relies on the database vendor to provide the underlying mechanism for generating ids. 

          
           
@Id
@GeneratedValue(strategy=GenerationType.AUTO)
private Long id;

SEQUENCE Strategy

The SEQUENCE strategy requires a database sequence object to be created before use. This is the most efficient of the strategies and is supported by all major databases. 

          
           
@Id
@GeneratedValue(strategy=GenerationType.SEQUENCE)
private Long id;

IDENTITY Strategy

The IDENTITY strategy uses an internal database mechanism, such as auto-increment fields in MySQL and SQL Server, to generate unique identifiers. 

          
           
@Id
@GeneratedValue(strategy=GenerationType.IDENTITY)
private Long id;

Conclusion

The GenerationType enum provides four strategies for generating entity ids: TABLE, AUTO, SEQUENCE, and IDENTITY. Each strategy has its own pros and cons, so choosing the right one depends on the database used and the application requirements.
