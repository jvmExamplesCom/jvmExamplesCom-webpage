javax.persistence.Table

Java Persistence API (JPA) is a Java application programming interface used to access relational database systems. A 

          
           
Table
is a core entity in JPA and is used to specify the table name in which an entity is stored or mapped. The 

          
           
Table
annotation is used to specify the primary table for the annotated entity. In this article we will take a look at the Table annotation, its attributes and how it is used in JPA.

Attributes of Table Annotation

The 

          
           
Table
annotation has the following attributes: 
  • name
    : This attribute is used to specify the name of the table. 
  • catalog
    : This attribute is used to specify the catalog of the table. 
  • schema
    : This attribute is used to specify the schema of the table. 
  • uniqueConstraints
    : This attribute is used to specify the set of unique constraints applied to the table.
  • indexes
    : This attribute is used to specify the set of indexes applied to the table.

Example of Table Annotation

Here is an example of the 

          
           
Table
annotation which specifies the name and schema of the table used to store the entities. 

          
           
@Entity
@Table(name = "Employee", schema = "APP")
public class Employee {
   // entity fields
}

Usage of Table Annotation

The 

          
           
Table
annotation is used to specify the table name in which an entity is stored or mapped. It is also used to specify the primary key column name of the table. The 

          
           
Table
annotation is used only once per entity class.
