Introduction to SpringApplicationBuilder

The 

          
           
org.springframework.boot.builder.SpringApplicationBuilder
is a utility class that provides convenient methods for configuring and launching a Spring application from the main() method. It simplifies the creation of a static application context and provides access to several common features such as logging and command line arguments. This article will discuss the basics of SpringApplicationBuilder, provide code examples, and explain how it can be used to create a basic Spring Boot application.

Creating a SpringApplicationBuilder

The most basic use of the 

          
           
SpringApplicationBuilder
is to create a new instance. This can be done in a few different ways. The simplest way to do this is with the default constructor: 

          
           
SpringApplicationBuilder builder = new SpringApplicationBuilder();
Alternatively, you can pass in an array of 

          
           
Class
objects to create different types of applications. For example, to create a web application, you can pass in the classes for the configuration and the entry point: 

          
           
SpringApplicationBuilder builder = new SpringApplicationBuilder(WebConfig.class, MyEntryPoint.class);
Finally, you can also specify additional sources when creating the builder: 

          
           
SpringApplicationBuilder builder = new SpringApplicationBuilder(MySource.class)
    .sources(WebConfig.class, MyEntryPoint.class);

Configuring the Application

Once you have created the builder, you can customize the application with various options. These options are set using the 

          
           
SpringApplicationBuilder
's 

          
           
configure
method. This method takes a 

          
           
Consumer<SpringApplicationBuilder>
, which can be used to configure the application: 

          
           
builder.configure(b -> {
     b.setBannerMode(Banner.Mode.CONSOLE);
     b.setApplicationName("My Application");
 });
The 

          
           
BannerMode
enum defines the different banner modes that can be used for an application. The 

          
           
setApplicationName
method sets a name that can be used by other components to identify the application.

Launching the Application

Once the application is configured, it can be launched with the 

          
           
SpringApplicationBuilder
's 

          
           
run
method. This method takes two arguments: the 

          
           
String[]
argument for the command line arguments, and an optional 

          
           
ApplicationContextInitializer
for setting up the application context before the application starts: 

          
           
ApplicationContext ctx = builder.run(args, initilizer);

Example

The following example demonstrates how to use the 

          
           
SpringApplicationBuilder
to create and launch a basic Spring Boot application using command line arguments and an initializer: 

          
           
public static void main(String[] args) {
    SpringApplicationBuilder builder = new SpringApplicationBuilder()
        .sources(MyConfig.class);

    ApplicationContext ctx = builder.run(args, ctx -> {
        System.setProperty("application.name", "My App");
    });
}
In this example, the application is configured with the 

          
           
MyConfig.class
configuration class and the command line arguments are used to set the application name.

Conclusion

The 

          
           
org.springframework.boot.builder.SpringApplicationBuilder
provides a convenient way to configure and launch a Spring application from the main() method. It simplifies the creation of a static application context and provides access to several common features such as logging and command line arguments. This article discussed the basics of SpringApplicationBuilder and provided code examples on how to use it.
