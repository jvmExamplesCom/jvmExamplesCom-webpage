Android WebSettings in Java

The Android platform provides the WebSettings class to manage settings associated with a WebView. It provides access to various preferences and settings for the WebView.

Basic Settings

To enable basic web settings, you can use the following example code: 

          
           
WebSettings webSettings = webView.getSettings();
webSettings.setJavaScriptEnabled(true);
webSettings.setSupportZoom(true);
webSettings.setBuiltInZoomControls(true);
This enables JavaScript, support for zooming, and built-in zoom controls.

User Agent

You can set a custom user agent string to be used when loading web content: 

          
           
WebSettings webSettings = webView.getSettings();
String ua = "Mozilla/5.0 (iPhone; CPU iPhone OS 8_0 like Mac OS X) AppleWebKit/600.1.3 (KHTML, like Gecko) Version/8.0 Mobile/12A4345d Safari/600.1.4";
webSettings.setUserAgentString(ua);

Cache Settings

To control caching, use the following methods: 

          
           
WebSettings webSettings = webView.getSettings();
webSettings.setAppCacheEnabled(true);
webSettings.setDatabaseEnabled(true);
webSettings.setDomStorageEnabled(true);
webSettings.setCacheMode(WebSettings.LOAD_DEFAULT);
This enables application, database and DOM storage caching, as well as setting the cache mode to be used. The available cache modes are: 

          
           
LOAD_DEFAULT,
LOAD_CACHE_ELSE_NETWORK,
LOAD_NO_CACHE,
LOAD_CACHE_ONLY

Miscellaneous Settings

You can also set miscellaneous settings such as minimum font size, text encoding, and more: 

          
           
WebSettings webSettings = webView.getSettings();
webSettings.setMinimumFontSize(12);
webSettings.setDefaultTextEncodingName("utf-8");
With the use of the Android 

          
           
WebSettings
class, you can customize and control the settings associated with a WebView.
