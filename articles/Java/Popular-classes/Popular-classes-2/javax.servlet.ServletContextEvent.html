javax.servlet.ServletContextEvent

The 
ServletContextEvent
is a class used in the Servlet API to provide an event that notifies an application about changes to the Servlet context of an application. It is part of the javax.servlet package.

What Does ServletContextEvent Do?

The ServletContextEvent is used to notify an application of events related to the Servlet context. A ServletContextEvent is sent to the ServletContextListeners registered with a web application when:

  • The web application is initialized.
  • The web application is destroyed.

Constructors

There are two constructors provided by ServletContextEvent class:

  • 
            
             
public ServletContextEvent(ServletContext sc)
    - creates a new ServletContextEvent initialized with the given Servlet context.
  • 
            
             
public ServletContextEvent(ServletContext source)
    - creates a new ServletContextEvent which is initialized with the given ServletContext source.

Methods

The ServletContextEvent class implements the following methods:

  • 
            
             
public ServletContext getServletContext()
    - returns the Servlet context associated with this event. 
  • 
            
             
public void setServletContext(ServletContext sc)
    - sets the ServletContext associated with this event.

Example Code

In the following example, we will create a simple ServletContextListener and define how it handles the ServletContextEvent.


          
           
public class MyServletContextListener implements ServletContextListener {

    public void contextInitialized(ServletContextEvent event) {
        System.out.println("ServletContext initialized.");
    }

    public void contextDestroyed(ServletContextEvent event) {
        System.out.println("ServletContext destroyed.");
    }
}

We will then register the listener with our web.xml file as follows:


          
           
<listener>
    <listener-class>
        com.example.MyServletContextListener
    </listener-class>
</listener>

Now when we deploy and start up the web application, we will see the following output on the console:


          
           
ServletContext initialized.

And when we undeploy and stop the web application, we will see the following output on the console:


          
           
ServletContext destroyed.
