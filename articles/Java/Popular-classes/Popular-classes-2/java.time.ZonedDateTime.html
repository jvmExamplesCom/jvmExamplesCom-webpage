java.time.ZonedDateTime

Java 8 introduced the 

          
           
java.time.ZonedDateTime
class, which provides a way of representing a date and time in respect to a certain load zone. These classes are immutable and thread safe, making them incredibly useful for avoiding race conditions when they are used in multi-threaded applications.

Example 1 - Create a ZonedDateTime

The following example shows how to create a 
ZonedDateTime
instance referencing the given time in the UTC time zone. 

          
           
ZonedDateTime zdt = ZonedDateTime.of(2020, Month.JULY, 10, 20, 15, 55, 0, ZoneOffset.UTC);
System.out.println(zdt);
The output of this example is: 

          
           
2020-07-10T20:15:55Z

Example 2 - Get Date from ZonedDateTime

The following example shows how to extract the date component from a 

ZonedDateTime
instance: 

          
           
LocalDate date = zdt.toLocalDate();
System.out.println(date);
The output of this example is: 

          
           
2020-07-10

Example 3 - Get Time from ZonedDateTime

The following example shows how to extract the time component from a 

ZonedDateTime
instance: 

          
           
LocalTime time = zdt.toLocalTime();
System.out.println(time);
The output of this example is: 

          
           
20:15:55

Example 4 - Get Zone from ZonedDateTime

The following example shows how to extract the zone component from a 

ZonedDateTime
instance: 

          
           
ZoneId zone = zdt.getZone();
System.out.println(zone);

Z

Example 5 - Combine Date and Time with Zone to ZonedDateTime

The following example shows how to combine a 
LocalDate
and 
LocalTime
with a time zone to create a 
ZonedDateTime
: 

          
           
LocalDate date = LocalDate.of(2020, Month.JULY, 10);
LocalTime time = LocalTime.of(20, 15, 55);
ZoneId zone = ZoneId.of("America/New_York");
ZonedDateTime zdt = ZonedDateTime.of(date, time, zone);
System.out.println(zdt);
The output of this example is: 

          
           
2020-07-10T20:15:55-04:00[America/New_York]
