org.junit.jupiter.api.BeforeEach

org.junit.jupiter.api.BeforeEach is an annotation used in the JUnit 5 testing framework to indicate that the annotated method should be executed before each test method in the current test class. This annotation helps developers with unit testing their code by making sure that no variables are left in an unexpected state between testmethod executions.

Syntax of BeforeEach Annotation

Below is the syntax for a BeforeEach annotation:


          
           
@BeforeEach
  void setUp() {
     // code executing before each test
  }

Example Usage

We will illustrate how to use the BeforeEach annotation with a simple example. Consider the following code for a Calculator class:


          
           
public class Calculator {
  private int result;

  public int add(int a, int b) {
     result = a + b;
     return result;
  }

  public int subtract(int a, int b) {
     result = a - b;
     return result;
  }
}

The following test class demonstrates the use of the BeforeEach annotation when testing this Calculator class:


          
           
public class CalculatorTest {

  private Calculator calculator;

  @BeforeEach
  void setUp() {
    calculator = new Calculator();
  }

  @Test
  void testAdd() {
    assertEquals(4, calculator.add(2, 2));
  }

  @Test
  void testSubtract() {
    assertEquals(3, calculator.subtract(5, 2));
  }
}

In this example, the Calculator instance is created before each test is executed. This ensures that the Calculator instance is always in its original state and no state is carried over from one test to another. This is particularly useful in cases where variables are not reset to their initial state, resulting in unexpected behavior.

Conclusion

The org.junit.jupiter.api.BeforeEach annotation is a useful feature of the JUnit 5 testing framework. It allows developers to ensure that variables are in their expected state before each test, ensuring that tests do not run incorrectly due to unintended changes in state. The example in this article showed how to use this annotation in a test class.
