Introduction to Java com.google.common.reflect.TypeToken

Java com.google.common.reflect.TypeToken helps developers to know the exact implementation of a given type at runtime, which is quite useful in generic programming. It supports the retrieval of the actual type-bedded class objects, allowing us to define specific methods and arguments without relying on casting and type information.

Example 1: Print Class Instance


          
           
import com.google.common.reflect.TypeToken;

public class Main {
   public static void main(String[] args) {
      TypeToken<String> token = TypeToken.of(String.class);
      System.out.println(token); 
   } 
}
Output: 

          
           
com.google.common.reflect.TypeToken<java.lang.String>

Example 2: Get the Parameterized Type


          
           
import com.google.common.reflect.TypeToken;
import java.util.List;

public class Main {
   public static void main(String[] args) {
      TypeToken<List<String>> token = new TypeToken<List<String>>(){};
      TypeToken.of(token.getType());
   } 
}
Output: 

          
           
com.google.common.reflect.TypeToken<java.util.List<java.lang.String>>

Example 3: Get the Supertype


          
           
import com.google.common.reflect.TypeToken;
import java.util.List;
import java.util.Collection;

public class Main {
   public static void main(String[] args) {
      TypeToken<List<String>> token = new TypeToken<List<String>>(){};
      TypeToken.of(token.getSupertype(Collection.class));
   } 
}
Output: 

          
           
com.google.common.reflect.TypeToken<java.util.Collection<java.lang.String>>

Conclusion

Java com.google.common.reflect.TypeToken helps developers to quickly access runtime type information and allows for various retrieval operations including the parameterized type, supertype and class instance. This is particularly helpful when working with generics and collections as it can help save time when casting and figuring out exact types.
