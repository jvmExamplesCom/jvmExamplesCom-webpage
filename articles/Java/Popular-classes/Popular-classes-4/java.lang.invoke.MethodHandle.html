Java.lang.invoke.MethodHandle

The Java.lang.invoke.MethodHandle class provides a way to create, inspect, and invoke methods and constructors of Java classes at run time.

Creating MethodHandles

MethodHandles can be created using the 

          
           
Lookup.findVirtual()
, 
Lookup.findStatic()
, and 

          
           
Lookup.findConstructor()
methods. The following example shows how to use these methods to create a MethodHandle for a method: 

          
           
//Create a MethodHandle for the "concat" method
MethodHandle mh = MethodHandles.lookup().findVirtual(String.class, "concat", 
    MethodType.methodType(String.class, String.class));
The above example creates a MethodHandle for the 
String.concat()
method. Here, the 

          
           
Lookup.findVirtual()
method is used, but if the method was static, then 
Lookup.findStatic()
should be used instead.

Once a MethodHandle is created it can be used to invoke the method using the 


          
           
MethodHandle.invoke()
method. This can be useful for quickly retrieving the result of a method without having to explicitly call it: 

          
           
//Invoke the "concat" method
String result = (String) mh.invoke("Hello ", "world!");
System.out.println(result); //Prints "Hello world!"

Inspecting MethodHandles

MethodHandles can also be used to inspect the underlying method or constructor. This can be done using the 


          
           
MethodHandle.reflectAs()
method. This method takes a 
Class
object as a parameter and returns an instance of the given class that represents the underlying method or constructor: 

          
           
//Inspect the underlying method
Method method = (Method) mh.reflectAs(Method.class);
System.out.println(method.getName()); //Prints "concat"
In this example, the 

          
           
MethodHandle.reflectAs()
method is used to get a 
Method
object which can be used to get information about the underlying method, such as its name.

Conclusion

The Java.lang.invoke.MethodHandle class allows for methods and constructors to be created, inspected, and invoked at run time. This can be useful for quickly inspecting or invoking methods without having to explicitly call them. Thus, it can make code more concise and easy to read.
