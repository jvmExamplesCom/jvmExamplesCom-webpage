java.sql.SQLFeatureNotSupportedException

The 

          
           
java.sql.SQLFeatureNotSupportedException
is an exception thrown when a feature is not supported by the driver connected to a database. This exception allows for proper error handling and notification of users when their application attempts to use a feature which is not supported by their current version of the driver.

When Is This Exception Thrown?

The 

          
           
java.sql.SQLFeatureNotSupportedException
is typically thrown when a java application attempts to access a feature which is not supported in the connected driver. This might be due to an old version of the driver or due to features not yet implemented and supported.

Examples

Let's look at some examples of when this exception might be thrown and how you can handle the exceptions with proper error handling.

Example 1: Unsupported Database Feature


          
           
try {
    // Access feature not supported by this version of the driver
    conn.createClob();
} catch (SQLFeatureNotSupportedException e) {
    System.err.println("This feature is not supported by the driver!");
    System.err.println("Error: " + e);
}

In this example, we are attempting to access a feature called 

createClob()
. If this feature is not supported by the connected driver, then a 

          
           
SQLFeatureNotSupportedException
will be thrown and caught. We can then print out an appropriate error message to the user.

try {
    // Access method not supported by this version of the driver
    conn.setFetchSize(0);

    System.err.println("Error: " + e);
}

In this example, we are attempting to access a method called 

setFetchSize()
. If this method is not supported by the connected driver, then a 

          
           
SQLFeatureNotSupportedException
will be thrown and caught. We can then print out an appropriate error message to the user.

Conclusion

java.sql.SQLFeatureNotSupportedException
