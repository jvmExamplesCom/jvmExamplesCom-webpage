Introduction to org.openjdk.jmh.annotations.Scope

org.openjdk.jmh.annotations.Scope, or simply Scope, is an annotation type used in the Java Microbenchmark Harness (JMH) for defining the scope of a benchmark. This annotation allows developers to easily apply scoping rules to their code to ensure consistent and repeatable performance measurements.

Scopes in JMH

In JMH, there are four different types of scopes: Thread, Group, Benchmark and Iteration. Each scope defines how a particular benchmark should be run, such as how many threads will be generated to run the benchmark, how many times the benchmark should be run, which threads should access the same data, and so on.

Thread Scope

The Thread scope specifies how many threads will be spawned to run the benchmark. The thread count is specified using the @Threads annotation, with a minimum of one thread being required.


          
           
@Threads(5)
public class MyBenchmark {
   // benchmark code
}

Group Scope

The Group scope allows multiple threads to access the same data. The group size is specified using the @Group annotation, with a minimum of one group being required. All threads assigned to the same group will have access to the same data.


          
           
@Group(3)
public class MyBenchmark {
   // benchmark code
}

Benchmark Scope

The Benchmark scope specifies how many times the benchmark should be run. The benchmark count is specified using the @RunTime annotation, with a minimum of one run being required.


          
           
@RunTime(10)
public class MyBenchmark {
   // benchmark code
}

Iteration Scope

The Iteration scope specifies how many iterations should occur per benchmark. The iteration count is specified using the @Iteration annotation, with a minimum of one iteration being required. If a benchmark has a time limit, the iterations can be limited to increase accuracy.


          
           
@Iteration(100)
public class MyBenchmark {
   // benchmark code
}

Conclusion

The org.openjdk.jmh.annotations.Scope annotation is used to define the scope of a benchmark in the Java Microbenchmark Harness. It allows developers to specify the number of threads, groups, benchmark runs, and iterations to ensure consistent and repeatable performance measurements.
