java.awt.datatransfer.UnsupportedFlavorException

java.awt.datatransfer.UnsupportedFlavorException is an exception that is thrown when an application requests or attempts to supply data in a format that is not supported by the underlying platform. This exception class is the root of all AWT data transfer flavor exception classes.

Examples

The following example is a code snippet that throws an UnsupportedFlavorException if the specified DataFlavor is not supported:


          
           
try {
  Object data = clipboard.getData(DataFlavor.stringFlavor);
} catch (UnsupportedFlavorException e) {
  // The specified flavor is not supported.
  System.out.println("Error: "+e);
}

In this example, an UnsupportedFlavorException is thrown when the data on the clipboard cannot be represented as a String.

The following example throws an UnsupportedFlavorException if the specified DataFlavor is not supported:


          
           
try {
  DataFlavor[] flavors = clipboard.getAvailableDataFlavors();
} catch (UnsupportedFlavorException e) {
  // The specified flavor is not supported.
  System.out.println("Error: "+e);
}

In this example, an UnsupportedFlavorException is thrown when the data on the clipboard cannot be represented with any of the available DataFlavors.

The following example demonstrates how to register a custom DataFlavor, and how to catch an UnsupportedFlavorException if the specified DataFlavor is not supported:


          
           
DataFlavor customFlavor = new DataFlavor("application/x-java-serialized-object;class=MyClass");
try {
  Object data = clipboard.getData(customFlavor);

  // The specified flavor is not supported.
