java.lang.management.MemoryUsage

MemoryUsage is a class from the Java Management Extensions (JMX) Java package. It is used to monitor and manage the memory usage of a given Java application. MemoryUsage is typically used for garbage collection and helping applications to calculate how much memory is currently being used and how much memory is available for use by the application.

Overview of MemoryUsage

MemoryUsage provides the following properties:
  • max: the maximum amount of memory that can be used by the application, in bytes
  • used: the amount of memory currently in use by the application, in bytes
  • committed: the amount of memory currently committed for usage by the application, in bytes
  • init: the initial amount of memory allocated for usage by the application, in bytes

Using MemoryUsage

MemoryUsage can be used to monitor the memory usage of an application. MemoryUsage provides methods for obtaining various information about the current memory usage of an application, including memory usage statistics and memory usage thresholds. These methods are listed below.

getMax()

This method returns the maximum amount of memory that can be used by the application, in bytes.


          
           
long maxMem = MemoryUsage.getMax();

getUsed()

This method returns the amount of memory currently in use by the application, in bytes.


          
           
long usedMem = MemoryUsage.getUsed();

getCommitted()

This method returns the amount of memory currently committed for usage by the application, in bytes.


          
           
long committedMem = MemoryUsage.getCommitted();

getInit()

This method returns the initial amount of memory allocated for usage by the application, in bytes.


          
           
long initMem = MemoryUsage.getInit();

setThresholds(...)

This method sets threshold values for memory usage. The first parameter of this method is the threshold level, which can be one of the following:

  • WARN: this threshold is triggered when the memory usage exceeds a high threshold value.
  • CRITICAL: this threshold is triggered when the memory usage exceeds a critical threshold value.

The second and third parameters of this method are the high and critical threshold values, respectively. These parameters specify the memory usage values (in bytes) at which a WARN or CRITICAL threshold is triggered.


          
           
MemoryUsage.setThresholds(MemoryUsage.WARN, 128000000, 256000000);

Example

The following example code uses the MemoryUsage class to monitor and manage memory usage of an application.


          
           
// Get the current memory usage
long memUsed = MemoryUsage.getUsed();

// Print the memory usage (in MB)
System.out.println("Memory Usage: " + (memUsed / 1024 / 1024) + "MB");

// Set the memory usage threshold levels 
MemoryUsage.setThresholds(MemoryUsage.WARN, 128000000, 256000000);

// Get the new memory usage
long newMemUsed = MemoryUsage.getUsed();

// Print the new memory usage (in MB)
System.out.println("New Memory Usage: " + (newMemUsed / 1024 / 1024) + "MB");
