javax.validation.constraints.NotEmpty

The javax.validation.constraints.NotEmpty annotation is used to declare a field, method or class as requiring a non-empty value. This means that fields marked with this annotation cannot be null, must contain elements if it is a collection type and the string should not contain only whitespace characters. In this article, we will look at examples of using the NotEmpty Annotation.

Example 1

In the following example, both the "name" and "description" fields are declared as not empty. If a null or empty value is assigned to either field, then a validation error will be thrown by the bean validator.


          
           
import javax.validation.constraints.NotEmpty;

public class Person{

    @NotEmpty
    private String name;
    
    @NotEmpty
    private String description;

}

Example 3

In this example, we create a custom validator. 

          
           
import javax.validation.ConstraintValidator;
import javax.validation.ConstraintValidatorContext;

public class NotEmptyValidator implements ConstraintValidator<NotEmpty,String> {
    @Override
    public boolean isValid(String value, ConstraintValidatorContext context) {
        return null != value &amp;&amp; !value.trim().isEmpty();
    }
}

Example 4

In this example, we apply our custom validator to a field.

import javax.validation.Payload;
