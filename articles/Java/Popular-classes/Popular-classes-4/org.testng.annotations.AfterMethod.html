org.testng.annotations.AfterMethod

The AfterMethod annotation of TestNG is used to define a method that needs to be invoked after each test method. This annotation is useful for cleaning up resources or methods which need to be run after each test case is executed.

Let's look at a simple example of how to use the AfterMethod annotation:


          
           
@Test
public void testMyName() {
  ...
} 

@AfterMethod
public void cleanup() {
  ...
}
In this example, the 
cleanup()
method will be executed after each test method. This can be used to free up resources or perform other necessary tasks after each test.

Attributes of AfterMethod Annotation

The AfterMethod annotation supports the following attributes:

  • alwaysRun
    : This tells TestNG to always invoke the method, even if any of the tests have been skipped.
  • description
    : This allows the user to provide a description for the method.
  • dependsOnMethods
    : This provides an array of methods that should be invoked before this method.
  • dependsOnGroups
    : This provides an array of groups whose methods should be invoked before this method.

Example of using AfterMethod Annotation

Let's look at a more complete example of how to use the AfterMethod annotation:


          
           
@Test (groups= {"group1", "group2"}) 
public void testGroup1() { 
   ...
} 

@Test (groups= {"group3"}) 
public void testGroup2() { 
   ...
} 

@AfterMethod (groups= {"group1"}) 
public void myCleanup() {
   ...
}
In this example, the 
myCleanup()
method will only be run after the 
testGroup1()
method. This is because it is declared as part of the 
group1
group. Therefore, this method will not be run after 
testGroup2()
, since it is not part of the 
group1
group.
