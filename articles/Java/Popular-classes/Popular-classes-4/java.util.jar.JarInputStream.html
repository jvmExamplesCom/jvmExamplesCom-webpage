Java.util.jar.JarInputStream

The Java.util.jar.JarInputStream class is a specialized InputStream for reading the contents of a JAR file, including the manifest and entries. It allows applications to read entry data sequentially. It extends the java.util.zip.ZipInputStream class.

Creating an Instance of JarInputStream

An instance of JarInputStream can be created by passing an instance of either FileInputStream or URLInputStream as an argument to the JarInputStream constructor. This creates an InputStream object which contains the contents of a JAR file.


          
           
// Create FileInputStream to read Jar File
FileInputStream fin = new FileInputStream("example.jar");

// Create JarInputStream 
JarInputStream jarInputStream = new JarInputStream(fin);

Reading the Contents of a JarInputStream

Once an instance of JarInputStream has been created, the contents of the JAR file can be read using the read() method. This method will return an instance of the ZipEntry class which contains information about the entry, such as its name, size, modification time, and type. The read() method will return null when all entries have been read.


          
           
// Initialize ZipEntry
ZipEntry zipEntry;

// Loop through all entries in the JarInputStream
while ((zipEntry = jarInputStream.read()) != null) {
    // Do something with the entry
}

Getting Manifest Information from a JarInputStream

The getManifest() method on the JarInputStream class returns an instance of the java.util.jar.Manifest class. This object contains information about the current JAR file, including its version, main class, and other attributes. Attributes can be accessed using their name as a String.


          
           
// Get the Manifest from the JarInputStream
Manifest manifest = jarInputStream.getManifest();

// Get the version attribute from the manifest
String version = manifest.getAttributes("version").getValue();

Closing a JarInputStream Object

When finished reading from a JarInputStream, it is important to close the object using the close() method. This ensures that all resources associated with the stream are released, allowing them to be reused by other parts of the program.


          
           
jarInputStream.close();
