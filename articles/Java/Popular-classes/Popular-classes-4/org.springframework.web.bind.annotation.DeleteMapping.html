org.springframework.web.bind.annotation.DeleteMapping

The

           
            
org.springframework.web.bind.annotation.DeleteMapping
annotation is used to specify that a method should be invoked when an HTTP DELETE request is sent to the specified URL. It can be used as an alternative to the @RequestMapping annotation in order to simplify the mapping process. This annotation can be used in either method signatures or class-level annotations.

Code example


          
           
@DeleteMapping("/users/{id}") 
public String deleteUser(@PathVariable("id") int id) {
    // delete user with the given id
    return "User deleted successfully";
}
The code example defines a method with 
@DeleteMapping
annotation. The annotation value specifies the path for which the method will be invoked when an HTTP DELETE request is sent. The method accepts an 
@PathVariable
annotation, which indicates that the path parameter needs to be passed in the url. This parameter will be used to identify the user that needs to be deleted.

Output

When an HTTP DELETE request is sent to the path specified in the 

@DeleteMapping
annotation, the method defined in the annotation will be invoked and the user specified in the path will be deleted. After deletion is successful, the method will return a string indicating the success of the deletion.
