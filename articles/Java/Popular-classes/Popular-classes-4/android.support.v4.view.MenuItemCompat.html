Android Support v4. View Menu ItemCompat

The Android Support Library v4 provides a lot of back-ported functionality to devices running Android 2.1 and above. One of the classes provided is MenuItemCompat which allows you to add and use a new ActionView class in your apps. This makes it convenient and easy to add a custom view to any menu item.

Usage

In order to use MenuItemCompat correctly, you will need to import the necessary classes.


          
           
import android.support.v4.view.MenuItemCompat;
import android.view.MenuItem;
import android.view.View;

You can then call the setActionView method on a MenuItem passed from the onCreateOptionsMenu() callback.


          
           
MenuItem item = menu.findItem(R.id.action_search);
View actionView = MenuItemCompat.getActionView(item);

Once the view has been set, you have complete control over what it contains. The MenuItemCompat class also provides some convenience methods for setting additional properties on that view.


          
           
MenuItemCompat.setShowAsAction(item, MenuItem.SHOW_AS_ACTION_IF_ROOM);
MenuItemCompat.setActionView(item, actionView);

For more complex use cases, the MenuItemCompat class provides access to the underlying implementation. This allows you to customize the behavior of the action view without having to rely on the default implementation.


          
           
// Get the support implementation
MenuItemImpl impl = (MenuItemImpl) MenuItemCompat.getMenuItemImpl(item);

// Set custom properties
impl.setShowAsAction(MenuItem.SHOW_AS_ACTION_IF_ROOM);
impl.setActionView(actionView);

Once you have configured your custom view, the view will be displayed when the menu item is clicked.

Example

Let's create a simple example of using MenuItemCompat to add a search box to the action bar. The first step is to create a new layout file to hold the search view.


          
           
<LinearLayout xmlns:android="http://schemas.android.com/apk/res/android"
              android:layout_width="match_parent"
              android:layout_height="match_parent">

    <SearchView
        android:id="@+id/search_view"
        android:layout_width="match_parent"
        android:layout_height="wrap_content" />

</LinearLayout>

Next, we will create the menu item and set the action view.


          
           
public boolean onCreateOptionsMenu(Menu menu) {
    getMenuInflater().inflate(R.menu.main, menu);

    // Get the search menu item
    MenuItem item = menu.findItem(R.id.action_search);

    // Get the action view
    View actionView = MenuItemCompat.getActionView(item);

    // Set the action view
    MenuItemCompat.setActionView(item, actionView);

    return true;
}

Finally, we will configure the action view to have the desired behavior.


          
           
// Get the support implementation
MenuItemImpl impl = (MenuItemImpl) MenuItemCompat.getMenuItemImpl(item);

// Set custom properties
impl.setShowAsAction(MenuItem.SHOW_AS_ACTION_IF_ROOM);
impl.setActionView(actionView);

Now when the menu item is clicked, the search view will be displayed. Thanks to the flexibility of the Android Support Library v4, you can easily add custom views to any menu item without having to write a lot of boilerplate code.
