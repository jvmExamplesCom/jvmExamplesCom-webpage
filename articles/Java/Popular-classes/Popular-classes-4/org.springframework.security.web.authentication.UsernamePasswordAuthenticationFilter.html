org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter

The 

          
           
UsernamePasswordAuthenticationFilter
is an important element of the Spring Security framework. It provides authentication mechanisms for authentication requests and validates submitted username/password credentials against a configured authentication manager. This filter allows users to log in with a username and password, while also providing additional security measures such as limiting login attempts, using secure passwords, and more. In this article, we will explore how the 

          
           
UsernamePasswordAuthenticationFilter
works and how it can be used to secure your web applications.

How It Works

When a user attempts to log in, the 

          
           
UsernamePasswordAuthenticationFilter
is responsible for verifying the submitted credentials. It does this by checking the username and password against a configured authentication manager. The authentication manager is responsible for authenticating users and returning an appropriate 
Authentication
object if the credentials are valid. If the authentication succeeds, the filter calls the successful authentication handler (such as 

          
           
SavedRequestAwareAuthenticationSuccessHandler
) to handle the request.

Configuration

The 

          
           
UsernamePasswordAuthenticationFilter
can be configured using XML or Java configuration. In its simplest form, the filter requires only a 

          
           
AuthenticationManager
bean to be configured: 

          
           
<beans:bean id="usernamePasswordAuthenticationFilter"
    class="org.springframework.security.web.authentication.UsernamePasswordAuthenticationFilter">
  <beans:property name="authenticationManager"
      ref="myAuthenticationManager" />
</beans:bean>
The 

          
           
UsernamePasswordAuthenticationFilter
can also be configured with various options such as limiting the number of login attempts, setting a minimum password length, and more.

Example

Let's look at an example that demonstrates how the 


          
           
UsernamePasswordAuthenticationFilter
can be used to secure a web application. In this example, we will create a simple web application with two pages: a login page and a protected page. The first step is to configure the 

          
           
UsernamePasswordAuthenticationFilter
in our Security Configuration class: 

          
           
@Configuration
@EnableWebSecurity
public class SecurityConfig extends WebSecurityConfigurerAdapter {

  // Other Configurations Here

  @Override
  protected void configure(HttpSecurity http) throws Exception {
    http
        .authorizeRequests()
            .antMatchers("/login").permitAll()
            .anyRequest().authenticated()
            .and()
        .formLogin()
            .loginPage("/login")
            .usernameParameter("username")
            .passwordParameter("password")
            .defaultSuccessUrl("/protected")
            .failureUrl("/login?error")
            .and()
        .logout()
            .logoutUrl("/logout")
            .logoutSuccessUrl("/login?logout")
            .and()
        .addFilterAt(
            new UsernamePasswordAuthenticationFilter(),
            UsernamePasswordAuthenticationFilter.class);
  }
}
Notice how the 

          
           
UsernamePasswordAuthenticationFilter
is added at the end. This ensures that the filter is applied to all requests.

Next, we will create the login page. This page should contain a form where the user can enter their username and password. The form should also have an action attribute set to the URL that the filter is configured to use:


          
           
<form action="/login" method="POST">
  <label for="username">Username:</label>
  <input type="text" name="username" />
  <label for="password">Password:</label>
  <input type="password" name="password" />
  <input type="submit" value="Log In" />
</form>

Finally, we will create the protected page. This page should be accessible only to logged in users:


          
           
<?php
  if($_SESSION['user'] != null) {
    echo 'Welcome back!';
  } else {
    echo 'Please log in.';
  }
?>
