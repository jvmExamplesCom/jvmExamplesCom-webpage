javafx.beans.value.ChangeListener

The 

          
           
javafx.beans.value.ChangeListener
interface is a part of the JavaFX API that is used when dealing with object properties, particularly when they are changed. It triggers callbacks when the value of a property changes, and allows for more complex logic handling of object changes. This article will explain how to use it, and will include many code examples.

Create a ChangeListener

Creating a ChangeListener is a relatively straightforward process. All you have to do is implement the ChangeListener interface, implement its abstract method "changed" and provide a custom implementation. In the following code example, we create an instance of ChangeListener and give it a title, then define its "changed" method.


          
           
ChangeListener<String> listener = new ChangeListener<String>() {
    @Override
    public void changed(ObservableValue<? extends String> source, String oldValue, String newValue) {
        System.out.println("Change detected!");
    }
};

Register a ChangeListener

A ChangeListener must be registered on a property in order to be triggered. The following code example shows how to register a ChangeListener on an instance of ReadOnlyObjectProperty:


          
           
ReadOnlyObjectProperty<String> property = ...;
property.addListener(listener);

The above code will register a ChangeListener on the property, and it will be triggered whenever the value of the property changes.

Triggering a ChangeListener

Now that you have a ChangeListener registered on a property, you can trigger it by changing the property's value. The following code example shows how to do this:


          
           
property.setValue("new value");

The above code will trigger the ChangeListener, and will print "Change detected!" to the console.

Removing a ChangeListener

Finally, you can remove a ChangeListener from a property by using the following code:


          
           
property.removeListener(listener);

The above code will unregister the ChangeListener, and it will no longer be triggered even if the property's value changes.
