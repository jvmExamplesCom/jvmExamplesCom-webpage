io.netty.channel.ChannelHandler.Sharable

The io.netty.channel.ChannelHandler.Sharable is an annotation which can be used to mark a ChannelHandler instance of type ChannelInboundHandler, ChannelOutboundHandler or CombinedChannelDuplexHandler as sharable - that is, safe to be added to several different pipelines concurrently.

Examples

Here's an example of a custom 
ChannelHandler
with the 
Sharable
annotation: 

          
           
@Sharable
public class MyChannelHandler extends ChannelInboundHandlerAdapter {
    // ...
}

Adding an instance of this handler to multiple pipelines is safe and will not cause any errors.

Usage

The 
Sharable
annotation should only be used on instances of 
ChannelHandler
s, such as 

          
           
ChannelInboundHandler
, 

          
           
ChannelOutboundHandler
or 

          
           
CombinedChannelDuplexHandler
. It is important to note that in order for a 
ChannelHandler
to be marked as 
Sharable
, all its state must remain constant over its lifetime. If the state is mutable, it should not be annotated with the 
Sharable
annotation. Furthermore, even if a 
ChannelHandler
is marked with 
Sharable
, its methods can still be invoked concurrently by different threads, so it is important to ensure thread-safety in the implementation of the handler.
