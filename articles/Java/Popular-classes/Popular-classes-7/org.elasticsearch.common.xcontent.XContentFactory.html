Introduction to org.elasticsearch.common.xcontent.XContentFactory

org.elasticsearch.common.xcontent.XContentFactory is a Java utility class provided by Elasticsearch used to serialize/deserialize objects into content forms, such as JSON and YAML.

Usage

XContentFactory can be used to serialize/deserialize objects into content forms. It provides several methods to do this:

  • 
            
             
createParser(InputStream)
    : creates XContentParser from the InputStream.
  • 
            
             
xContentType(InputStream)
    : returns the XContentType of the input stream.
  • jsonBuilder()
    and 
    yamlBuilder()
    : creates instances of XContentBuilder for building XContent from scratch.
  • 
            
             
createGenerator(OutputStream, XContentType)
    : creates an XContentGenerator for the OutputStream.
  • 
            
             
createGenerator(Writer, XContentType)
    : creates an XContentGenerator for the Writer.

Examples

Creating an XContentBuilder

XContentBuilder can be used to build your own XContent from scratch:


          
           
XContentBuilder builder = XContentFactory.jsonBuilder();
builder.startObject().field("field1", "value1").field("field2", 42).endObject();
This will create a JSON object 

          
           
{"field1":"value1","field2":42}
. To write it out, use the 
builder.bytes()
method, which returns the ContentBuilder as an array of bytes that can be sent to the OutputStream.

Creating an XContentParser

XContentParser can be used to parse an existing XContent object:


          
           
InputStream inputStream = new ByteArrayInputStream("{\"field1\":\"value1\",\"field2\":42}".getBytes());
XContentParser parser = XContentFactory.xContent(XContentType.JSON).createParser(inputStream);
And then use the 
parser.nextToken()
method to process each token in the content.

Creating an XContentGenerator

XContentGenerator can be used to serialize objects in an XContent form:


          
           
XContentGenerator generator = XContentFactory.yamlGenerator(outputStream);
generator.writeStartObject().writeFieldName("field1").writeString("value1").writeFieldName("field2").writeNumber(42).writeEndObject();
This will create a YAML object 

          
           
field1: value1\nfield2: 42
. To write it out, use the 
generator.flush()
method which flushes the data to the OutputStream.

Conclusion

org.elasticsearch.common.xcontent.XContentFactory provides several useful methods for working with XContent and can be a great asset when dealing with complex data. With its easy-to-use interface, it can simplify the process of serializing/deserializing objects.
