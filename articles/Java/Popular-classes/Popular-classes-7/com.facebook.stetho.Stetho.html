Introduction to com.facebook.stetho.Stetho

com.facebook.stetho.Stetho is an Android debugging bridge that allows developers to better inspect the state of their applications. This library makes it easier to debug, diagnose and understand the behavior of the app, which in turn can lead to a better user experience.

This article will provide an overview of the Stetho library, a detailed run-through of how to use it and some code examples to get you started.

Overview of Stetho

Stetho provides developers with an easy way to access all of the internal state of their application. By using Stetho, developers can view and interact with a running Android application via Chrome’s Developer Tools. This allows them to understand and debug UI issues, database contents, network requests and more.

The library also offers developers an extensible plugin model that allows them to add additional functionality. There are currently several plugins available, such as the Facebook Network Inspector, which allows developers to monitor HTTP requests, and the Chrome DevTools Console, which provides interactive debugging capabilities.

Other features of the library include:
  • Access to SQLite database files
  • Viewing of network requests
  • Real-time view of network response
  • Interactive console for runtime evaluation
  • Automatic “hot reload” of code changes

Using Stetho

Using Stetho is very straightforward and requires only a few steps. First, you’ll need to add the library as a dependency to your project. This can be done by adding the following line to your app’s build.gradle file:


          
           
compile 'com.facebook.stetho:stetho:1.5.0'

Once this is done, you’ll then need to initialize the bridge in your application’s onCreate method:


          
           
Stetho.initializeWithDefaults(getApplicationContext());

That’s it! Now you can open Chrome and type the URL "chrome://inspect" into the address bar to access the developer tools. You should see your device listed under the “Devices” tab.

Now you can start exploring the internal state of your application. You can view the current view hierarchy, look at the database contents, monitor HTTP requests, inspect live-running objects and so on.

Example Code

To help illustrate how to use Stetho, we’ll walk through a simple example. We’ll write an application that allows users to query a database of books and display the results. We’ll then use Stetho to inspect the data.

First, we’ll need to create the database. We’ll use SQLite for this example:


          
           
SQLiteDatabase db = openOrCreateDatabase("books.db", MODE_PRIVATE, null);
db.execSQL("CREATE TABLE IF NOT EXISTS books (title TEXT, author TEXT)");
db.execSQL("INSERT INTO books VALUES ('The Hobbit', 'J.R.R. Tolkien')");
db.execSQL("INSERT INTO books VALUES ('Alice's Adventures in Wonderland', 'Lewis Carroll')");
db.execSQL("INSERT INTO books VALUES ('The Catcher in the Rye', 'J.D. Salinger')");

Now we’ll write a function that queries the database and returns the results:


          
           
public List<Book> queryBooks(String keyword) {
    List<Book> books = new ArrayList<>();
    Cursor cursor = db.rawQuery("SELECT * FROM books WHERE title LIKE ? OR author LIKE ?", 
        new String[]{"%" + keyword + "%", "%" + keyword + "%"});
    while (cursor.moveToNext()) {
        Book book = new Book(
            cursor.getString(cursor.getColumnIndex("title")),
            cursor.getString(cursor.getColumnIndex("author"))
        );
        books.add(book);
    }
    return books;
}

Finally, we’ll need to make the Stetho library aware of the database. We can do this by adding the following snippet anywhere in our code:


          
           
Stetho.defaultInspectorModulesProvider(getApplicationContext())
  .provideDatabaseDriver(new AndroidSqliteDatabaseDriver(getApplicationContext(), db, "books.db"));

Now we can open Chrome, type the URL "chrome://inspect" into the address bar, select our device and open the database tab. We’ll see our database listed and can explore the contents.

Stetho is a powerful debugging tool that enables developers to quickly diagnose and debug applications. It can be used to inspect database contents, view the UI hierarchy, monitor network requests and more. As shown in the example above, it is very easy to set up and use.
