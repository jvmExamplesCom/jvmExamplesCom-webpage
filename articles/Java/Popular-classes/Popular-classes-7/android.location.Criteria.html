Android Location Criteria

Android Location Criteria is an API for Android applications to control the active location provider. An application can use this API to request or query the availability of a certain location provider on the device.

Overview of Location Criteria API

The Location Criteria API enables an application to specify criteria for a desired location provider and thereby allow the system to select the best provider that meets the criteria. This API also allows an application to query the available location providers on the device. The API includes the following:

  • getBestProvider()
    : Returns the name of the best provider based on the criteria.
  • getProviders()
    : Returns a list of all enabled providers.
  • setAccuracy()
    : Sets the accuracy criteria.

    : Sets the power criteria.

setAltitudeRequired()
    : Sets the altitude criteria.
  • 
            
             
setBearingRequired()
    : Sets the bearing criteria.
  • setSpeedRequired()
    : Sets the speed criteria.

Using Location Criteria API

The following example shows how to use the Location Criteria API to request for a location provider. The application sets the accuracy and power requirement criteria and then requests for the best location provider that meets these criteria.


          
           
LocationCriteria criteria = new LocationCriteria();
criteria.setAccuracy(Criteria.ACCURACY_FINE);
criteria.setPowerRequirement(Criteria.POWER_HIGH);
String bestProvider = criteria.getBestProvider();
