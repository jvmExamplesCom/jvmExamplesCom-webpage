javax.xml.bind.annotation.XmlEnumValue

javax.xml.bind.annotation.XmlEnumValue is an annotation that is used to map java enum constants to XML representation. This annotation is used to mark fields of the enum type to be processed by a JAXB annotated object. The XmlEnumValue annotation allows developers to specify custom values for specific enum constants in the target XML schema. By default, the XML representation of an enum constant is its name. However, by using XmlEnumValue, it is possible to override this default mapping and assign a different value to some or all enum constants.

Example 1: Simple enum with XmlEnumValue


          
           
@XmlEnum
public enum Day {
   SUNDAY("SUN"),
   MONDAY("MON"),
   TUESDAY("TUE"),
   WEDNESDAY("WED"),
   THURSDAY("THU"),
   FRIDAY("FRI"),
   SATURDAY("SAT");
 
   // ...
 
   private final String abbreviation;
   Day(String abbreviation) {
      this.abbreviation = abbreviation;
   }
 
   @XmlEnumValue
   public String getAbbreviation() {
      return abbreviation;
   }
}

The above code defines a simple enum type called “Day”. Each enum constant is assigned an abbreviation as an argument to its constructor. The field “abbreviation” is marked with the XmlEnumValue annotation, which tells the JAXB binding framework to use the value of this field as the XML representation of the associated enum constant.

Example 2: Overriding Default Mapping


          
           
@XmlEnum
public enum Color {
    GREEN,
    BLUE,
    RED;
 
    // ...
 
   @XmlEnumValue("Gr")
   GREEN,
   @XmlEnumValue("Bl")
   BLUE,
   @XmlEnumValue("Rd")
   RED;
}

The above code defines an enum called “Color” with three constants: GREEN, BLUE, and RED. All three constants are marked with the XmlEnumValue annotation. The annotation specifies custom XML representations for each enum constant: Gr for GREEN, Bl for BLUE, and Rd for RED. By using the annotation, it is possible to override the default mapping and assign custom values to enum constants.

The above example demonstrates how XmlEnumValue can be used with nested enums. The outer enum Fruit contains two constants APPLE and ORANGE. The inner enum Citrus contains two constants LEMON and LIME. Each constant is marked with the XmlEnumValue annotation, and the annotation specifies custom XML representations for each enum constant.

Conclusion

The XmlEnumValue annotation provides a powerful and flexible way to override the default mapping between Java enums and their XML representation. It can be used to specify custom values for specific enum constants, or to apply a single mapping to all constants in an enum. In addition, the annotation can be used with both top-level and nested enums.
