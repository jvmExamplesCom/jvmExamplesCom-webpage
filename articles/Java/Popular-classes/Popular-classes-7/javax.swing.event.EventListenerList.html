Introduction to javax.swing.event.EventListenerList

javax.swing.event.EventListenerList is an interface provided in the Java programming language for registering and delivering EventListener lists. Java provides a specific package for events known as java.awt.event. It contains numerous classes and methods to set up and respond to events. This includes the ActionListener, WindowListener, and MouseListener.

Often times it is necessary to have multiple listeners listen to the same event. For situations like this, we can use the EventListenerList class from javax.swing.event. This class enables us to combine multiple listeners into a single object, thus allowing us to register and deliver multiple listeners at once.

Example Code

Let’s take a look at how to use EventListenerList. First, let’s define an interface to represent our action listeners. This action listener will be invoked when a button is clicked.


          
           
public interface ActionListener {
    void actionPerformed(ActionEvent e);
}

Now, let’s define a class which implements the above action listener interface. This class will be used to register the action listeners.


          
           
public class MyActionListener implements ActionListener {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("Action performed!");
    }
}

Next, let’s create an instance of EventListenerList. This is where we will store the list of action listeners.


          
           
EventListenerList listenerList = new EventListenerList();

Now, let’s add some action listeners to our listener list.


          
           
listenerList.add(ActionListener.class, new MyActionListener());
listenerList.add(ActionListener.class, new MyActionListener());
listenerList.add(ActionListener.class, new MyActionListener());

Finally, let’s iterate over the list of listeners and invoke the actionPerformed method on each of them.


          
           
for (ActionListener listener : listenerList.getListeners(ActionListener.class)) {
    listener.actionPerformed(null);
}

The output of the above program will be “Action performed!” three times, since we have added three action listeners.

Conclusion
