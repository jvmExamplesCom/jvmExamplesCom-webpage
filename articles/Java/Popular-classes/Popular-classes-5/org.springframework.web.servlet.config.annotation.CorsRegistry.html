Java - org.springframework.web.servlet.config.annotation.CorsRegistry

The 

          
           
org.springframework.web.servlet.config.annotation.CorsRegistry
class is a part of the Spring Framework that is used to enable Cross-Origin Resource Sharing (CORS) with mapping-based configuration. It is a powerful tool for creating secure applications that can handle requests from different origins and respond accordingly.

Cross-Origin Resource Sharing (CORS) supports secure communications between web clients and servers by providing a way for a server to accept requests from a different domain than its own. Without CORS, web clients are restricted to making requests only to the same origin as the server. This can prevent malicious actors from attacking a website or service.

Enabling Cors

To enable CORS in your application, you must add a CorsRegistry to your servlet context. This will allow you to configure the allowed origins and methods. 


          
           
@Configuration
public class WebConfig implements WebMvcConfigurer
{
    @Override
    public void addCorsMappings(CorsRegistry registry)
    {
        registry.addMapping("/api/**")
            .allowedOrigins("http://example.org")
            .allowedMethods("GET", "POST");
    }
}

This will enable CORS for any request with a path beginning with 

/api/
, allowing the origin 
http://example.org
, and the methods 
GET
and 
POST
. Any other requests from other origins or with other methods will be denied.

Adding Headers

You can also add headers to your responses to provide additional information. For example, to add an 


          
           
Access-Control-Allow-Origin
header to all responses, you can do: 

          
           
registry.addMapping("/**")
    .allowedOrigins("*")
    .exposedHeaders("Access-Control-Allow-Origin");
You can even set multiple headers: 

          
           
registry.addMapping("/**")
    .allowedOrigins("*")
    .exposedHeaders("Access-Control-Allow-Origin", 
                    "Access-Control-Allow-Methods");

Conclusion

CORS helps provide a secure way to communicate between clients and servers, while allowing the flexibility to access resources from different origins. The 


          
           
org.springframework.web.servlet.config.annotation.CorsRegistry
class allows you to easily configure CORS in your application.
