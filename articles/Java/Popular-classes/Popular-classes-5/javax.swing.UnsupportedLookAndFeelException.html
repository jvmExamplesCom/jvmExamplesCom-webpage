javax.swing.UnsupportedLookAndFeelException

javax.swing.UnsupportedLookAndFeelException is an exception thrown when a program attempts to set the look and feel (L&F) of a Swing component but the L&F is not supported on the current platform. It is a subclass of java.lang.Exception and is thrown whenever a UIManager.setLookAndFeel call canâ€™t find the specified L&F class, or when a LookAndFeel class obtained via UIManager.getLookAndFeel returns false when asked if supported on this platform.

Example usage

try {
    UIManager.setLookAndFeel("javax.swing.plaf.nimbus.NimbusLookAndFeel");
} catch (ClassNotFoundException | InstantiationException | IllegalAccessException | UnsupportedLookAndFeelException e) {

}

javax.swing.UnsupportedLookAndFeelException: Nimbus [....] not supported on this platform

Handling

When handling an UnsupportedLookAndFeelException, it is generally best practice to simply log the exception before re-throwing it - any action to reset the look and feel must be done manually, since the look and feel that was attempted to be set was not supported on the current platform anyway.
