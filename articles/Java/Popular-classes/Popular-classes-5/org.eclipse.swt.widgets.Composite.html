org.eclipse.swt.widgets.Composite

The org.eclipse.swt.widgets.Composite class is an abstract superclass that provides a foundation for all the SWT Widgets. It is a special container which contains and organizes other widgets, is intended to be extended by subclasses, and can also be used directly as a widget container.

Creating A Composite

To create a composite, you must first instantiate the Composite class. You can pass in a parent Composite, or a Display in the constructor. You must also specify a style, which is an integer sum of zero or more of the Style flags.


          
           
Composite composite = new Composite(parent, style);

Add Widgets to a Composite

Once you have instantiated a Composite, you can add widgets to it. This is done by instantiating appropriate widget classes, and passing the composite to the constructor.


          
           
Button button = new Button(composite, style);
Text text = new Text(composite, style);

Layout Managers

By default, the Composite does not specify how its child widgets should appear. For that, you need to set a Layout Manager for the Composite. There are several different types of layout managers available in the SWT library.


          
           
composite.setLayout(new RowLayout(style));

Listeners

You can add listeners to the Composite in order to receive events from user actions, such as selection of a widget, or mouse clicks. This is done by instantiating the appropriate listener class and adding it to the Composite.


          
           
SelectionListener listener = new SelectionAdapter() {

    public void widgetSelected(SelectionEvent e) {
        // Handle event
    }
};

button.addSelectionListener(listener);

Dispose

When you are done using the Composite and its child widgets, you should call the dispose() method on the Composite. This will properly clean up any resources associated with the Composite, such as listeners and layouts.


          
           
composite.dispose();
