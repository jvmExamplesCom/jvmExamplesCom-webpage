javax.persistence.Inheritance

Java Persistence API (JPA) provides the abstract concept of an inherited entity, allowing developers to create single table inheritance hierarchies. The 


          
           
javax.persistence.Inheritance
annotation is used to specify the inheritance strategy for an entity class hierarchy. The following code snippet shows how to use the 

          
           
javax.persistence.Inheritance
annotation: 

          
           
@Entity
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
public class Person {

   //... 

}
In the above example, the 
@Inheritance
annotation specifies that the Person class should be mapped to a single database table. The 

          
           
javax.persistence.Inheritance
annotation supports the following 
InheritanceType
values: 

          
           
1. SINGLE_TABLE
2. JOINED
3. TABLE_PER_CLASS

InheritanceType.SINGLE_TABLE

The 
SINGLE_TABLE
inheritance strategy maps the entity hierarchy into a single table in the database. All classes in the hierarchy are mapped to the same table and the 
discriminator
column is used to differentiate between different classes.

For example, consider the following entity hierarchy:


          
           
@Entity
@Inheritance(strategy = InheritanceType.SINGLE_TABLE)
@DiscriminatorColumn(name = "type", discriminatorType = DiscriminatorType.STRING)
public class Person {

   //... 

}

@Entity
@DiscriminatorValue("employee")
public class Employee extends Person {

   //... 

}

@Entity
@DiscriminatorValue("customer")
public class Customer extends Person {

   //... 

}

The above entity hierarchy will be mapped to the following database table:


          
           
person 
+--------------------+-------------+-------+
|       type         |    name     | age   |
+--------------------+-------------+-------+
| employee           | John Smith  |  25   |
+--------------------+-------------+-------+
| customer           | Jane Doe    |  32   |
+--------------------+-------------+-------+

InheritanceType.JOINED

The 
JOINED
inheritance strategy maps the entity hierarchy into multiple database tables. Each class in the hierarchy is mapped to a separate table and a join is used to retrieve data from multiple tables.

For example, consider the following entity hierarchy:


          
           
@Entity
@Inheritance(strategy = InheritanceType.JOINED)
public class Person {

   //... 

}

@Entity
public class Employee extends Person {

   //... 

}

@Entity
public class Customer extends Person {

   //... 

}

The above entity hierarchy will be mapped to the following database tables:


          
           
person

|  id   |  name   | age  |
+-------+---------+------+
|   1   | John Smith  | 25  |
+-------+---------+------+
|   2   | Jane Doe   | 32  |
+-------+---------+------+

employee
+-------+---------+--------+
|  id   |  salary | bonus  |
+-------+---------+--------+
|   1   |   10000 |  1000  |
+-------+---------+--------+

customer
+-------+-----------+
|  id   |  credit   |
+-------+-----------+
|   2   |    100    |
+-------+-----------+

InheritanceType.TABLE_PER_CLASS

The 
TABLE_PER_CLASS
inheritance strategy maps the entity hierarchy into multiple database tables, one table for each class in the hierarchy. All columns for a particular class are stored in the table for that class.

For example, consider the following entity hierarchy:
