org.codehaus.jackson.annotate.JsonProperty

The 

          
           
org.codehaus.jackson.annotate.JsonProperty
annotation is used to indicate a field used in serialization and deserialization of JSON data when using the Jackson library. It is an annotation used mainly for Java classes but is also supported in other languages.

Usage

The 
@JsonProperty
annotation can be used on a class field, method or constructor parameter to indicate that it should be used as part of the serialization and deserialization of JSON data. This annotation can also be used to customize the mapping of JSON data to an object.

Example


          
           
@JsonProperty("name")
private String name;

public MyClass(@JsonProperty("name") String name){
    this.name = name;
}

public String getName(){
    return this.name;
}
In the above code example, the 
@JsonProperty
annotation has been used to indicate that the field 
name
is used during serialization and deserialization of JSON data, and that it should be mapped to the key 
name
. In addition, the annotation is used in the constructor to indicate that the parameter should also be used in the JSON serialization and deserialization process.

Customization

The 
@JsonProperty
annotation supports various types of customizations, such as the ability to specify the name of the property, the type of the property, and the access level. For example: 

          
           
@JsonProperty(value="name", access=Access.READ_WRITE)
private String name;
In the above code example, the 
@JsonProperty
annotation has been used to customize the mapping of the JSON data to an object by specifying the key name, as well as the access level (which in this case is read-write). This allows for more control over how the data is mapped and how it is accessed.
