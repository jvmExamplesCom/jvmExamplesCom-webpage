javax.persistence.EmbeddedId

The javax.persistence.EmbeddedId is a special type of persistent field used to map composite values in a JPA entity. It is used when an entity has a composite primary key and is typically used when the primary key is composed of two or more fields. The EmbeddedId annotation allows a composite primary key class to be treated as a single logical key value.

Example

The following example shows an entity with a composite primary key consisting of two fields:

public class BankAccount {

    @EmbeddedId
    private BankAccountPK primaryKey;

    // Attributes and getters/setters, etc...

}

@Embeddable
public class BankAccountPK implements Serializable {

    private String accountNumber;
    private int bankCode;

    // Getters/Setters

}

BankAccount

accountNumber

bankCode

@Embeddable

To use the 
EmbeddedId

@EmbeddedId

EmbeddedId
