Introduction to org.springframework.context.support.ResourceBundleMessageSource

The org.springframework.context.support.ResourceBundleMessageSource class is a specialized message source that allows access to resource bundles using specified basenames. It enables the application developer to create easily internationalized applications without writing any additional code for loading and delivering messages from the resource bundles.

Creating an Instance of ResourceBundleMessageSource

You can create an instance of ResourceBundleMessageSource with a single line of code: 

          
           
ResourceBundleMessageSource msgSource = new ResourceBundleMessageSource();

Setting Basename

The basename parameter is used to set the locale specific resource bundle that it should look up. To set the basename, you can use the setBasename method like this: 

          
           
msgSource.setBasename("messages");
This will tell the ResourceBundleMessageSource to look up the messages resource bundle in the classpath root.

Getting Messages

Once the ResourceBundleMessageSource is set up, you can get the messages from the resource bundle like this: 

          
           
String message = msgSource.getMessage("message.key", null, Locale.getDefault());
The first parameter is the message key, the second parameter is the array of parameters to be replaced in the message and the third parameter is the locale object. The Locale.getDefault() will retrieve the default locale from the JVM.

Caching

By default, the ResourceBundleMessageSource caches all the retrieved messages. This behavior can be changed by setting the cacheSeconds parameter. For example, to cache the messages for 1 hour: 

          
           
msgSource.setCacheSeconds(3600);
