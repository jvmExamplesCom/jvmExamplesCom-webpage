Java org.elasticsearch.common.transport.TransportAddress

Java's org.elasticsearch.common.transport.TransportAddress is an abstract base class that allows for network communication between two different endpoints. It can be used for both local and remote communication, and provides a uniform way for applications to establish connections, regardless of the underlying transport mechanism.

Creating the TransportAddress

The most basic way to create a TransportAddress is with its constructor:


          
           
TransportAddress address = new TransportAddress("127.0.0.1", 9300);

This will create a new TransportAddress instance using the localhost IP address ("127.0.0.1") and port number (9300) as parameters. This particular address is suitable for server-client communication over the default Elasticsearch transport protocol, which uses TCP/IP.

Setting Basic Address Options

address.setHostname("example.com");
address.setPort(80);

Using the TransportAddress

Once the TransportAddress has been configured, it can then be used for communication purposes. The simplest way to do this is to use the send() method:


          
           
byte[] data = { 0x12, 0x34, 0x56, 0x78 };

This will send the specified data to the configured address, and receive back a TransportResponse object with the response. The data sent and the response received can both be byte arrays, though it is also possible to use more complex datatypes such as JSON or XML.
