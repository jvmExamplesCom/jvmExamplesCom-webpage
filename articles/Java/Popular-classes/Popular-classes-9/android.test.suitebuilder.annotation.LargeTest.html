Android.test.suitebuilder.annotation.LargeTest

The Android.test.suitebuilder.annotation.LargeTest annotation indicates that a test is classified as a large test and may be executed on the instrumentation framework but only when explicitly requested. This annotation is part of the Android Testing Support Library.

Usage

The @LargeTest annotation can be used to mark a test method as a large test. For example:


          
           
@LargeTest
public void myLargeTest() {
   ...
}

All tests marked with @LargeTest will be run as large tests. By default, all large tests will have a timeout of 10 seconds. This can be changed by passing a different value as an argument to the @LargeTest annotation:


          
           
@LargeTest(timeout=20)
public void myLargeTest() {
   ...
}

Example

Here is an example showing how to use the @LargeTest annotation in your test code. We will create two methods, one that is marked as a small test, and one that is marked as a large test. We will then create a TestSuite and run the tests:


          
           
@SmallTest
public void mySmallTest() {
   ...
}

@LargeTest
public void myLargeTest() {
   ...
}

public static TestSuite getTestSuite() {
   TestSuite suite = new TestSuite();
   suite.addTest(mySmallTest());
   suite.addTest(myLargeTest());
   return suite;
}

public static void main(String[] args) {
   TestRunner.run(getTestSuite());
}

Running this code with the instrumentation framework will run both the small test and the large test. The large test will be executed at the end of the test run, after all of the small tests have completed.
