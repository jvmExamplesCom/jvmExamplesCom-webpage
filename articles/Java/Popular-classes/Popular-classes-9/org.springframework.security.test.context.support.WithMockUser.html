The org.springframework.security.test.context.support.WithMockUser Annotation

The org.springframework.security.test.context.support.WithMockUser is an annotation used within integration tests for applications secured by Spring Security. It allows test writers to designate a "user" with certain roles and authorities who can be utilized for integration testing. As such, it allows the application to be tested for a variety of user scenarios in order to ensure the desired functionality is available for those different user scenarios.

Overview

The org.springframework.security.test.context.support.WithMockUser annotation allows developers to specify a "user" with marked roles and authorities that can be utilized during an integration test. The annotation will then create a MockUser object and inject it as part of the test setup. This MockUser object will then serve as the current user for the rest of the integration test.

Declaring the Annotation

The org.springframework.security.test.context.support.WithMockUser annotation must be declared with one or more parameters. The parameters available for the annotation are:
  • username - The username for the mock user.
  • password - The password for the mock user.
  • authorities - A list of authorities granted to the mock user.
  • roles - A list of roles granted to the mock user.
