androidx.annotation.FloatRange: Introduction

AndroidX FloatRange is an annotation library for Android applications that allows developers to annotate a method parameter, field or method return value with a reference range of acceptable float values. This can be used to provide guarantees to the API user that any argument provided won't be outside of the specified range.

Annotation Syntax

The following syntax is used to declare the FloatRange annotation: 

          
           
@FloatRange(from = 0.0) annotatedParameter
The 'from' keyword denotes the minimum allowed value in the range and the 'to' denotes the maximum allowed value. If only the 'from' keyword is provided, it indicates that any float value greater than or equal to the provided value is accepted.

Examples

The following examples demonstrate how to use the FloatRange annotation: 

          
           
// Annotates a field as having values between 0.0 and 1.0
 @FloatRange(from = 0.0, to = 1.0) private float field; 

 // Annotates a method parameter to have a value >= 0.0
 public void foo(@FloatRange(from = 0.0) float param) {
 } 

 // Annotates a method return value to have a value <= 10.0
@FloatRange(to = 10.0) public float bar() {
    return 5.0f;
}

Compile-Time Checks

The FloatRange annotation enables compile-time checks to ensure that arguments passed to a method are within the specified range. The following example demonstrates this: 

          
           
public void foo(@FloatRange(from = 0.0, to = 10.0) float param) {
 // do something
}

Conclusion

In conclusion, the FloatRange annotation provides a powerful way to specify a range of acceptable float values and to enforce those boundaries at both compile-time and runtime. This can be used to ensure the correctness of application code and to provide guarantees to API users.
