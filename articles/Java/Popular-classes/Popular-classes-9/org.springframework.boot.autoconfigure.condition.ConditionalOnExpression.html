org.springframework.boot.autoconfigure.condition.ConditionalOnExpression

The 

          
           
org.springframework.boot.autoconfigure.condition.ConditionalOnExpression
class enables developers to configure a bean with the annotation 

          
           
@ConditionalOnExpression
. This annotation is used to indicate that a bean will be created based upon a SpEL expression defined within the annotation. With this annotation, developers can determine whether or not a bean should be created based on the predefined expressions.

Example:


          
           
import org.springframework.boot.autoconfigure.condition.ConditionalOnExpression;
import org.springframework.stereotype.Component;

@Component
@ConditionalOnExpression("#{systemProperties['environment'] == 'dev'}")
public class MyBean {
    // code here
}
In the example above, we can see how 

          
           
@ConditionalOnExpression
is used. The annotation is defined on the class and indicates that the bean will only be created if the SpEL expression evaluates to true. In this case, we are evaluating a system property called 'environment'. If the value is equal to 'dev', then the bean will be created.

Configuration using Java Properties

We can also configure the SpEL expression using a Java properties file. This can be done by using the 

          
           
@ConditionalOnExpression
annotation and passing it a string which can be resolved against a properties file. 

          
           
@ConditionalOnExpression("${myproperty:defaultValue}")
In this case, the expression is looking for a property called "myproperty" in the properties file. If it is not found, then it will use the default value of "defaultValue".

More Complex Expressions

@ConditionalOnExpression
can also handle more complex expressions. It allows for the use of logical operators such as AND (&&), OR (||), and NOT (!). This allows for more powerful expressions which can be used to determine whether or not the bean should be created.
