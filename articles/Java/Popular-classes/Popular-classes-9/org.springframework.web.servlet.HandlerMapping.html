Introduction - org.springframework.web.servlet.HandlerMapping

The 

          
           
org.springframework.web.servlet.HandlerMapping
is an interface implemented by objects responsible for mapping requests to handler objects that perform the requested action. It provides a consistent way of mapping URLs to handler objects both within and across applications. This interface must be implemented by any class that wishes to provide URL-to-handler mapping capabilities in a web application.

Anatomy of Handler Mappings

A handler mapping consists of the following components:
  • A 
    RequestHandler
    object, which is responsible for handling each request.
  • A 
    
            
             
HandlerExecutionChain
    , which contains the set of components that will execute the request.
  • A 
    HandlerInterceptor
    object, which can intercept a request before it reaches the handler.
  • A 
    
            
             
HandlerExceptionResolver
    , which can resolve exceptions thrown by the handler or any of its components.

Request Handler

The 

          
           
RequestHandler
is responsible for handling the request. It must implement the 

          
           
HandlerAdapter
interface and must be configured to handle a specific request. The handler adapter must also be able to determine if the handler can handle the request or not.

Handler Execution Chain

Once the request handler has been determined, the handler mapping then creates a 

          
           
HandlerExecutionChain
, which contains the set of components that will execute the request. This chain can include any number of components, such as 

          
           
HandlerInterceptor
or 

          
           
HandlerExceptionResolver
, as well as the actual request handler itself. The order in which the components are executed is determined by the handler mapping.

Handler Interceptors

The 

          
           
HandlerInterceptor
objects can intercept a request before it reaches the handler. They can be used to perform authentication, authorization, or other types of pre-processing. An interceptor can be configured to execute before or after the request handler, or both.

Handler Exception Resolver

The 

          
           
HandlerExceptionResolver
objects can be used to resolve exceptions thrown by the handler or any of its components. These exceptions can include server errors, client errors, or any other type of exception. A handler exception resolver can be configured to resolve any type of exception.

SimpleUrlHandlerMapping
