javax.transaction.TransactionManager

The javax.transaction.TransactionManager class provides the interface for an application server to manage transactions on behalf of applications. It is a core component of the Java Transaction API (JTA) and provides an API for managing transaction boundaries between components. The API allows for local and global transactions to be managed in the same way, allowing for flexibility and scalability.

Methods

The javax.transaction.TransactionManager interface provides several methods for creating, suspending and resuming transactions. These methods are:

  • begin() – This method starts a new transaction. The transaction associated with the thread is returned.
  • commit() – This method commits the transaction associated with the current thread.
  • rollback() – This method rolls back the transaction associated with the current thread.
  • getTransaction() – This method returns the transaction associated with the current thread.
  • suspend() – This method suspends the current transaction and returns the transaction object associated with the thread.
  • resume(Transaction t) – This method resumes the transaction associated with the given transaction object.

Examples

Below is an example of how to use the javax.transaction.TransactionManager class:


          
           
import javax.transaction.Transaction;
import javax.transaction.TransactionManager;

// Create a new transaction manager instance
TransactionManager tm = createTransactionManager();
 
// Start a new transaction
Transaction t = tm.begin();
 
try {
    // Do some work in the context of the transaction
    doWork();
 
    // Commit the transaction
    tm.commit(t);
} catch (Exception e) {
    // Rollback the transaction
    tm.rollback(t);
}

Transaction t = tm.suspend();

    // Do some work outside of the transaction context
    doWork();
 
    // Resume the transaction
    tm.resume(t);
} catch (Exception e) {
    // Rollback the transaction
