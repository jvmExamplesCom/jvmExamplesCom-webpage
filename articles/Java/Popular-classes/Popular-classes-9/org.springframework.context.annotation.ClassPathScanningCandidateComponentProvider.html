org.springframework.context.annotation.ClassPathScanningCandidateComponentProvider

The 

          
           
org.springframework.context.annotation.ClassPathScanningCandidateComponentProvider
provides an easy way of searching within the Java class path in order to find classes loaded by various ways.

A few common usage scenarios are (with code examples):

Getting Class References


          
           
ClassPathScanningCandidateComponentProvider scanner = new ClassPathScanningCandidateComponentProvider();
Set<BeanDefinition> classes = scanner.findCandidateComponents("package.prefix");
for (BeanDefinition bd : classes) {
   Class type = Class.forName(bd.getBeanClassName());
   // do something with type...
}
The example above creates a new 

          
           
ClassPathScanningCandidateComponentProvider
, then scans for any 
BeanDefinition
within the package with the 
package.prefix
. Then, it iterates over the found definitions and gets their associated class reference.

Detecting Classes Annotated with @MyAnnotation


          
           
ClassPathScanningCandidateComponentProvider scanner = 
    new ClassPathScanningCandidateComponentProvider();
scanner.addIncludeFilter(new AnnotationTypeFilter(MyAnnotation.class));
Set<BeanDefinition> classes = scanner.findCandidateComponents("package.prefix");
for (BeanDefinition bd : classes) {
   Class type = Class.forName(bd.getBeanClassName());
   // do something with type...
}

The code block above is much the same as the previous example, but also adds a filter which checks to see if the candidate class has the 

@MyAnnotation
annotation applied.

Using Custom Filters and Type Filters


          
           
ClassPathScanningCandidateComponentProvider scanner = 
   new ClassPathScanningCandidateComponentProvider();
scanner.addIncludeFilter(new CustomFilter());
scanner.addIncludeFilter(new AssignableTypeFilter(MyType.class));
Set<BeanDefinition> classes = scanner.findCandidateComponents("package.prefix");
for (BeanDefinition bd : classes) {
   Class type = Class.forName(bd.getBeanClassName());
   // do something with type...
}
The example above introduces two new filters, a 
CustomFilter
and a 

          
           
AssignableTypeFilter
. The 
CustomFilter
allows you to define your own criteria for filtering classes, while the 

          
           
AssignableTypeFilter
can be used to ensure that the found classes extend or implement a certain type (in this case 
MyType
).

Conclusion

The 

          
           
ClassPathScanningCandidateComponentProvider
provides an easy way of searching through the Java Class Path in order to find classes loaded from various sources.

It can be used with multiple filters in order to further refine the search criteria, and is an invaluable tool when searching for classes with specific annotations or types.
