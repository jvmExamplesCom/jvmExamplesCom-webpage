ResponseBody annotation in Java

The 

          
           
@ResponseBody
annotation is one of the Spring MVC annotations used to map a controller method return value to a response body. It can be used with any of the HTTP methods such as GET, POST, PUT, and DELETE but it only commonly used with GET and POST methods.

Syntax

The syntax of the 

          
           
@ResponseBody
annotation is very simple. It is declared as a class-level annotation in the @Controller annotation declaration: 

          
           
@Controller
public class MyController {

    @RequestMapping("/myendpoint")
    @ResponseBody
    public String getMyData() {
        // method implementation here
    }
}

Usage

The usage of the 

          
           
@ResponseBody
annotation depends on the return type of the controller method. If the return type is a 

          
           
String
, the 

          
           
@ResponseBody
will generally just wrap the string with an HTML response. For example, if the method returns 

          
           
Hello World!
, the browser will render the HTML 

          
           
<b>Hello World!</b>
: 

          
           
@RequestMapping("/myendpoint")
@ResponseBody
public String getMyData() {
    return "Hello World!";
}
If the return type is an object, the 

          
           
@ResponseBody
annotation will serialize the object into JSON or XML depending on the content type of the request. This can be useful for exposing endpoints that can be consumed by other applications. For example, if the method returns an instance of 

          
           
Person
, the following may be returned in JSON format: 

          
           
@RequestMapping("/myendpoint")
@ResponseBody
public Person getMyData() {
    Person person = new Person("John", "Smith");
    return person;
}
The output of this may be a JSON like this: 

          
           
{
    "name": "John",
    "surname": "Smith"
}

Conclusion

The 

          
           
@ResponseBody
annotation is a powerful tool for mapping controller methods to response bodies. Its syntax is simple and it can be used to serialize objects into JSON or XML depending on the content type of the request.
