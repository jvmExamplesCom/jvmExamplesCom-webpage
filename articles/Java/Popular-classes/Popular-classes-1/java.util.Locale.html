java.util.Locale

The java.util.Locale class is a class in the Java Platform that represents a geographical, political, or cultural region. It is used to represent a specific language or country, and helps display locale-sensitive information such as dates, times, number formats, currencies, and measurements. The Locale class can be used with many other classes in the Java Platform to determine appropriate regional settings.

Creating a Locale

There are two constructors for creating a Locale: 

          
           
Locale(String language)
Locale(String language, String country)

For example, if you wanted to create a Locale for the US in English, you would use the following:


          
           
Locale l = new Locale("en", "US");

You can also create a Locale from a string representation of the combination of language, country, and variant using the following constructor:


          
           
Locale(String language, String country, String variant)

For example, you can create a Locale for the UK in English and a specific variant like this:


          
           
Locale l = new Locale("en", "GB", "POSIX");

Using the Locale Class

The Locale class offers several methods for retrieving information about the locale. The most commonly used are:


          
           
String getCountry()
String getLanguage()
String getVariant()

These methods will return the respective values that were used when creating the Locale. For example, if you created a Locale for the US, you would get the following result:


          
           
Locale l = new Locale("en", "US");
System.out.println(l.getCountry()); // US
System.out.println(l.getLanguage()); // en
System.out.println(l.getVariant()); // null

The Locale class also provides methods for retrieving the default locale for the current user. The most commonly used for this purpose are:


          
           
static Locale getDefault()
static Locale[] getAvailableLocales()
The 
getDefault()
method returns the default Locale for the current user. The 

          
           
getAvailableLocales()
method returns an array of all the supported Locales on the current system.

The Locale class also provides methods for formatting dates, times, numbers, and currencies based on the current locale. The most commonly used for this purpose are:


          
           
NumberFormat getCurrencyInstance()
DateFormat getDateInstance()
DateFormat getDateTimeInstance()
NumberFormat getNumberInstance()
NumberFormat getPercentInstance()

These methods will return an instance of the respective class that is configured with the settings of the current Locale. For example, if you wanted to format a date with the current locale, you could do the following:


          
           
Locale l = Locale.getDefault();
DateFormat df = l.getDateInstance();
String formattedDate = df.format(new Date());
System.out.println(formattedDate);

The output of this code would be a string representation of the current date in the current locale.

Conclusion

The java.util.Locale class is a useful tool for dealing with regional settings in the Java platform. It can be used to create Locales, retrieve information about them, and format dates, times, numbers, and currencies according to the current locale.
