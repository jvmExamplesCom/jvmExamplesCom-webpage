javax.persistence.Column

Introduction

javax.persistence.Column is a class part of the Java Persistence API that provides an annotation-based mapping for fields and properties of persistent entities in Java. It was introduced as part of JPA 1.0 and is used to map database columns to object-oriented fields.

Annotations Overview

The javax.persistence.Column class includes a wide range of annotations to help customize the mapping of fields and properties to database columns. This includes field-level annotations for things like column name, length, type, and other properties as well as support for more complex annotations.

Basic Annotation Examples

The most basic usage of javax.persistence.Column is to provide a field-level annotation to indicate the name of the column that should be used when persisting the field or property data to the database. For example:


          
           
@Column(name="field_name")
private String fieldName;

By default, the field name will be used for the column name unless a different name is specified.

The @Column annotation also supports setting the length of the column via the length attribute, which can be used to limit the size of the data being stored:


          
           
@Column(name="field_name", length=50)
private String fieldName;

In addition, the @Column annotation supports setting the type of the column via the columnDefinition attribute:


          
           
@Column(name="field_name", columnDefinition="VARCHAR(50)")
private String fieldName;

The type of the column needs to be compatible with the type of the field or property being persisted.

Advanced Annotation Examples

The javax.persistence.Column class also provides a number of advanced annotations for customizing the mapping of fields and properties to database columns. For example, the nullable attribute can be used to indicate whether null values are allowed for the column:


          
           
@Column(name="field_name", nullable=false)
private String fieldName;

The unique attribute can be used to indicate that the value of the column must be unique in the database:


          
           
@Column(name="field_name", unique=true)
private String fieldName;

And the insertable and updatable attributes can be used to indicate whether the column should be included in insert or update statements:


          
           
@Column(name="field_name", insertable=false, updatable=false)
private String fieldName;

Conclusion

javax.persistence.Column provides a powerful annotation-based approach for mapping fields and properties of entities to database columns. Its annotations can be used to customize the columns used for persisting the entity data, from basic configurations like column name, length, and type to more advanced options like nullability, uniqueness, and inclusion in insert/update statements.
