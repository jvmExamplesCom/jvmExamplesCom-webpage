org.springframework.context.annotation.ComponentScan

Spring Framework provides ComponentScan annotation to search for components and configure them in the ApplicationContext. This annotation enables developers to directly specify the packages which contain the required components, instead of explicitly configuring the individual classes. The annotation also helps to automatically register beans with the application context.

Configuration

The ComponentScan annotation is used to define which packages need to be scanned for including the beans for the application context. The annotation can be given at the class level or at the package level.

Example 1: Class Level Annotation

In the following example, the ComponentScan annotation is used to scan all the components in the package com.example.app. 

          
           
@ComponentScan("com.example.app")
public class AppConfig {
  // bean definitions go here
}

Example 2: Package Level Annotation

The following example shows how to use ComponentScan at the package level. It scans all the components in the com.example.app and its subpackages. 

          
           
@ComponentScan(basePackages = "com.example.app")
package com.example;
import org.springframework.context.annotation.ComponentScan;

Example 3: Filtering Components

It is possible to filter the components that need to be registered with the application context while scanning the packages. The following example shows how to use the includeFilters attribute of component scan. 

          
           
@ComponentScan(basePackages = "com.example.app", 
    includeFilters = { @Filter(type = FilterType.ASSIGNABLE_TYPE, 
                              value = CalculatorService.class) })
package com.example;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.FilterType;
import org.springframework.context.annotation.ComponentScan.Filter;

Example 4: Excluding Components

It is also possible to exclude certain components from being registered with the application context. The following example shows how to use the excludeFilters attribute of component scan. 

          
           
@ComponentScan(basePackages = "com.example.app", 
    excludeFilters = { @Filter(type = FilterType.ANNOTATION, 
                              value = Deprecated.class) })
package com.example;
import org.springframework.context.annotation.ComponentScan;
import org.springframework.context.annotation.FilterType;
import org.springframework.context.annotation.ComponentScan.Filter;

Example 5: Using BasePackageClasses Attribute

The basePackageClasses attribute of ComponentScan can be used to scan for components within a given package. The following example shows how to use this attribute. 

          
           
@ComponentScan(basePackageClasses = {com.example.CalculatorService.class})
package com.example;
import org.springframework.context.annotation.ComponentScan;

Conclusion

The ComponentScan annotation allows developers to specify the packages which contain the components and configure them in the ApplicationContext. It can also be used to filter the components that need to be scanned and configure selective components.
