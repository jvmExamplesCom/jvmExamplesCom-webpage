javax.servlet.ServletContext

The 

          
           
javax.servlet.ServletContext
interface provides access to the configuration and context information of the web application. It is an object that holds specific information about the servlet and also acts as a global resource provider. The 
ServletContext
object can be used to get handle to other objects within the web application. It provides access to attributes and init parameters. Also, files related to the web application can be accessed via this object.

Creating a ServletContext

A 
ServletContext
object can be obtained in two ways:
  • By passing it from a servlet to another servlet using the 
    getServletContext()
    method in the 
    ServletConfig
    object.
  • By using the 
    
            
             
ServletContextListener
    . This interface provides a hook for loading and initializing an application context for a servlet.

Methods of the ServletContext

The 
ServletContext
object provides access to the files in the application. It also provides access to variables stored in the application which can be global or local. 

          
           
String version = getServletContext().getInitParameter("version");
Map<String, Object> appAttr = getServletContext().getAttributes();
String filePath = "/WEB-INF/hello.txt";
InputStream inputStream = getServletContext().getResourceAsStream(filePath);
The methods exposed by the 
ServletContext
interface are: 
  • getContextPath()
    : Returns the context path of the web application.
  • getAttributeNames()
    : Returns the names of the attributes stored in the application.
  • 
            
             
getInitParameterNames()
    : Returns the names of the init parameters stored in the application.
  • 
            
             
getRealPath(String path)
    : Returns the real path from the specified virtual path.
  • 
            
             
getResourceAsStream(String path)
    : Returns an input stream for the specified resource.
  • 
            
             
setAttribute(String name, Object value)
    : Stores an attribute in the application.

Example

In the following example, a servlet that prints out the version of the application and the list of attributes is created: 


          
           
import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

public class PrintAppInfoServlet extends HttpServlet {

    public void doGet(HttpServletRequest req, HttpServletResponse res) 
        throws ServletException, IOException {

        ServletContext ctx = getServletContext();
        String appVersion = ctx.getInitParameter("version");
        PrintWriter out = res.getWriter();
        out.println("Application Version: " + appVersion);
        out.println("Attributes:");
        for (String attrName : ctx.getAttributeNames())
            out.println(attrName);
    }
}

This servlet prints out the version of the application which is defined as an init parameter in the 

web.xml
file. It also prints out the names of the attributes that have been stored in the application.
