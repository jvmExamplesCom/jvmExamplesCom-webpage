What is Java.util.concurrent.Callable?

Java.util.concurrent.Callable is an interface that enables a program to define a task to be performed and return the result of the task. In other words, the Callable interface gives the ability to define a task and return a result to the caller in the form of a Future object.

Example Usage

The following example shows how a Callable task can be created and submitted to an ExecutorService: 

          
           
ExecutorService executor = Executors.newFixedThreadPool(2);

Callable<String> task = new Callable<String>(){
	public String call(){
		return "Hello world";
	}
};

Future<String> future = executor.submit(task);
executor.shutdown();
In this example, we create a thread pool with 2 threads and submit a Callable task. The returned Future object can be used to check if the task is done, wait for the task to finish and to access the result.

Callable vs Runnable

While both Callable and Runnable are used to create tasks, there is one major difference between them. The Callable interface is parameterized with the return type of the task while the Runnable interface has no return type. This makes it possible to return a value to the caller when using the Callable interface.

Getting Result from Future

Once the Callable task is submitted to ExecutorService, the Future object returns an object which represents the result of the task. To get the result, the get() method needs to be invoked. The get() method will block until the Callable is finished. 

          
           
Future<String> future = executor.submit(task);
String result = future.get();
Alternatively, the get() method can also be passed a timeout argument to specify the maximum amount of time the program should wait for the result. 

          
           
Future<String> future = executor.submit(task);
String result = future.get(10, TimeUnit.SECONDS);
If the task is not finished within the specified timeout, a TimeoutException is thrown.

Cancelling Tasks

The Future object of a Callable task can also be used to cancel the task. To cancel the task, the cancel() method needs to be invoked. This method takes two boolean arguments, the first one specifying if the task should be interrupted if it is currently executing and the second one specifying if the task should be interrupted even if it has already started executing but has not yet finished. 

          
           
Future<String> future = executor.submit(task);
boolean cancelled = future.cancel(true, true);
If the task was successfully cancelled, the method will return true, otherwise it will return false.

Conclusion

Java.util.concurrent.Callable provides a powerful way to create and execute tasks. It enables programs to define tasks and return a result in the form of a Future object. The Future object can be used to monitor the progress of the task, check if the task is done and to retrieve the result of the task. It also provides methods to cancel the task if necessary.
