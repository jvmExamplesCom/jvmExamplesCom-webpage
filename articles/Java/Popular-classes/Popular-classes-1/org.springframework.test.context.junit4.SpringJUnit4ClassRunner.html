org.springframework.test.context.junit4.SpringJUnit4ClassRunner

The SpringJUnit4ClassRunner is a custom extension of JUnit's BlockJUnit4ClassRunner that provides functionality of the Spring TestContext Framework to standard JUnit tests by means of the TestContextManager and associated support classes and annotations. To achieve this, the SpringJUnit4ClassRunner class executes each test method within a TestContext .

Code Examples

Below are several examples of code that illustrate the use of the SpringJUnit4ClassRunner.


          
           
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration("/test-context.xml")
public class MyTests {
 
    @Test
    public void testSomething() {
        // test code here
    }
}

The above example illustrates a basic JUnit test that is run with the SpringJUnit4ClassRunner. Here, we have specified a configuration file for the application context.


          
           
@RunWith(SpringJUnit4ClassRunner.class)
@ContextConfiguration(locations = {"/test-context1.xml", "/test-context2.xml"})
public class MyTests {
 
    @Test
    public void testSomething() {
        // test code here
    }
}

This example is similar to the one above, except that it allows you to specify multiple configuration files for the application context.


          
           
@RunWith(SpringJUnit4ClassRunner.class)
@ContextHierarchy({
    @ContextConfiguration("/test-context1.xml"),
    @ContextConfiguration("/test-context2.xml")
})
public class MyTests {
 
    @Test
    public void testSomething() {
        // test code here
    }
}

This example is similar to the one above, except that it uses the @ContextHierarchy annotation instead of @ContextConfiguration. This enables you to specify multiple configuration files for the application context in a hierarchical manner.
