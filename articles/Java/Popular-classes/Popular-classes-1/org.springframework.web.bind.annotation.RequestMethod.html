org.springframework.web.bind.annotation.RequestMethod

The org.springframework.web.bind.annotation.RequestMethod class provides enumerations which represent the HTTP request methods which can be used in the use-case of a Spring application.

Methods

There are 8 enum members in RequestMethod:
  • GET: requests a specified resource from the server.
  • HEAD: similar to get, but without the body.
  • POST: requests that the server accept the data enclosed in the body of the request message for storage.
  • PUT: requests that the enclosed entity be stored under the supplied URI.
  • PATCH: used to apply partial modifications to a resource.
  • DELETE: requests that the specified resource be deleted.
  • OPTIONS: used to describe the communication options for the target resource.
  • TRACE: used to perform a message loop-back test along the path to the target resource.

Example

Here is an example of how to use the RequestMethod in a controller layer:


          
           
@Controller
public class MyController {

    @RequestMapping(value = "/hello", method = RequestMethod.GET)
    public String sayHello() {
        return "Hello World!";
    }
}

In this example, when a GET request is sent to the "/hello" URL in the application, the sayHello() method is executed and the "Hello World!" string is returned as the response.
