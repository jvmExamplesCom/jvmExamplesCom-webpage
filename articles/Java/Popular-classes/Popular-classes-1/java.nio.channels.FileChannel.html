java.nio.channels.FileChannel

java.nio.channels.FileChannel is an open, thread-safe channel for accessing the file system. It provides read, write and manipulate capabilities for files in the file system. FileChannel gives access to a file's content on a byte-by-byte basis, and can be used to map files into memory.

Creating a FileChannel

A FileChannel must be created before any operations can be done on it. To create a FileChannel, first create a FileInputStream, FileOutputStream, or RandomAccessFile, and then invoke its getChannel() method. For example: 


          
           
RandomAccessFile aFile = new RandomAccessFile("data/nio-data.txt", "rw");
FileChannel inChannel = aFile.getChannel();

The above code snippet creates a RandomAccessFile which is associated with the file data/nio-data.txt. The 

rw
mode argument to RandomAccessFile indicates that it should be opened for reading and writing. Finally, the FileChannel can be obtained by invoking the getChannel() method of RandomAccessFile.

Reading from a FileChannel

Once a FileChannel is created, we can read data from it. Data is read from a FileChannel into a Buffer. A Buffer is a container for data, and it has methods to manipulate the data. In order to read a file into a Buffer, we must first create a Buffer, then read the data into it. For example:


          
           
ByteBuffer buf = ByteBuffer.allocate(48);
int bytesRead = inChannel.read(buf);

The above code snippet creates a ByteBuffer of size 48 bytes and reads the data from the FileChannel into the buffer. The number of bytes read is returned and stored in the variable bytesRead.

Writing to a FileChannel

Data can also be written to a FileChannel. Writing is done by transferring data from a Buffer to the FileChannel. For example:


          
           
String newData = "New String to write to file..." + System.currentTimeMillis();
 
ByteBuffer buf = ByteBuffer.allocate(48);
buf.clear();
buf.put(newData.getBytes());
 
buf.flip();
 
while(buf.hasRemaining()) {
     inChannel.write(buf);
}

The above code snippet creates a ByteBuffer containing the string "New String to write to file..." followed by the current time. The data is then transferred from the Buffer to the FileChannel by writing it out. It is important to call the flip() method of the Buffer before writing, so that the internal pointer of the Buffer is set to the beginning of the data.

Closing a FileChannel

When you are done with the FileChannel, it must be closed to free up system resources. Closing a FileChannel is very simple:


          
           
inChannel.close();

This closes the FileChannel and frees up any system resources it was using.

Conclusion

FileChannel gives Java developers access to the file system on a byte-by-byte basis, and provides read, write and manipulate capabilities to files. To use FileChannel, first create a RandomAccessFile, FileInputStream, or FileOutputStream. Then invoke the getChannel() method to obtain the FileChannel. With the FileChannel, data can be read from and written to the file system. When you are done with the FileChannel, it must be closed to free up system resources.
