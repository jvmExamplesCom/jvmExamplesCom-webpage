Introduction to java.util.TreeMap

The 
java.util.TreeMap
class is a member of the Java Collections Framework, which allows you to store and manipulate data in a very efficient manner. It implements the 
SortedMap
interface, which provides an ordering of its key/value pairs based on the keys' natural order or the order defined by a 
Comparator
provided at the time the map is created. A 
TreeMap
is a great choice when you need a flexible and powerful way to store and retrieve key/value pairs. It provides fast retrieval times for both small and large sets of data. Additionally, it provides several methods for more advanced operations, such as merging, copying, and sorting.

How TreeMap Works

Under the hood, the 
TreeMap
uses a Red-Black tree to back the structure of the map. This type of tree offers a balance between the time complexity for the basic operations (insert, contains, remove) and the number of items stored in the map. By using a tree data structure, the 
TreeMap
can offer fast lookups, no matter the size of the data set. The 
TreeMap
also implements the 
NavigableMap
interface, which allows it to provide several methods for navigating through the elements of the map. These methods allow us to easily retrieve the first element, last element, and elements by range.

Creating a TreeMap

You can create an empty 
TreeMap
with the default constructor: 

          
           
TreeMap<String, Integer> numbers = new TreeMap<>();

This will create an empty map that will hold String keys and Integer values.

If your keys implement the 
Comparable
interface, you can specify the ordering of the elements in your map. For example, if you have a class 
Employee
that implements 
Comparable
, you can create a map to store them like this: 

          
           
TreeMap<Employee, String> employees = new TreeMap<>();

This will create a map that stores Employee keys and String values, and the elements will be ordered according to the natural ordering of the Employee objects.

If you need a specific ordering for the elements but the keys don't implement the 

Comparable
interface, you can use a 
Comparator
. You can pass a 
Comparator
to the constructor of the map to define the ordering: 

          
           
Comparator<String> ignoreCase = Comparator.comparing(String::toLowerCase);
TreeMap<String, Integer> numbersByLowerCase = new TreeMap<>(ignoreCase);

This creates a map that stores String keys and Integer values, and the ordering is based on the lowercase version of the strings.

Adding Elements to a TreeMap

You can add elements to the map using the 
put()
method. This method takes a key and a value and adds a new element to the map: 

          
           
numbers.put("one", 1);
employees.put(new Employee("John"), "manager");
numbersByLowerCase.put("FIVE", 5);
It's important to note that the 
put()
method won't add duplicate keys to the map. If you try to add an element with the same key as an existing key, the existing element will be replaced.

Retrieving Elements from a TreeMap

You can get the value associated with a given key using the 

get()
method. This method takes a key and returns the associated value, or 
null
if there is no element with the given key: 

          
           
Integer one = numbers.get("one");
String job = employees.get(new Employee("John"));
Integer five = numbersByLowerCase.get("FIVE");

Removing Elements from a TreeMap

You can remove an element from the map using the 
remove()
method. This method takes a key and removes the element associated with it: 

          
           
numbers.remove("one");
employees.remove(new Employee("John"));
numbersByLowerCase.remove("FIVE");

Iterating Through a TreeMap

You can loop through the elements of a 
TreeMap
using the 
entrySet()
method. This method returns a 
Set
of 
Map.Entry
objects which you can loop through: 

          
           
Set<Map.Entry<String, Integer>> numbersEntries = numbers.entrySet();
for (Map.Entry<String, Integer> entry : numbersEntries) {
    System.out.println(entry.getKey() + " = " + entry.getValue());
}
// Output:
// one = 1

java.util.TreeMap
class. We saw how to define a 
TreeMap
and how to add, retrieve, and remove elements from it. Finally, we saw how to iterate through a 
TreeMap
's entries.
