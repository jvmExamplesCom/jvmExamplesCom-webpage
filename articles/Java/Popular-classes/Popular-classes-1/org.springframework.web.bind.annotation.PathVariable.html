org.springframework.web.bind.annotation.PathVariable

The PathVariable annotation is a type of annotation provided by the Spring Framework that is used to access and bind values from the URI to a method parameter. This annotation allows developers to access parts of the URI that are dynamic as opposed to static and store it in a variable.

Information

Here is an example of the PathVariable annotation being declared on a method parameter. The @PathVariable annotation is applied to the method's parameter and declares that the value of the parameter should be taken from the URI.


          
           
@GetMapping("/example/{id}")
public void getExample(@PathVariable int id) {
   // do something with "id"
}

This annotation has two optional attributes: name and required. The name attribute allows you to specify the name of the variable in the URI, instead of using the method parameter's name. The required attribute allows you to specify whether the value must be present in the URI for the request to succeed.

Examples

Example 1: Retrieving a Value from the URI

In this example, the @PathVariable annotation is used to retrieve a value from the URI and assign it to an int variable.


          
           
@GetMapping("/example/{id}")
public void getExample(@PathVariable int id) {
   System.out.println("The ID is: " + id);
}
If the URI is 
/example/123
, the output of this example would be: 

          
           
The ID is: 123

Example 2: Specifying a Name for the Variable

In this example, we are specifying the name of the variable in the URI. Notice how we have used the name attribute of the @PathVariable annotation to specify that the value in the URI should be bound to the 

exampleId
variable. 

          
           
@GetMapping("/example/{exampleId}")
public void getExample(@PathVariable(name = "exampleId") int id) {
   System.out.println("The ID is: " + id);
}

Example 3: Require a Value in the URI

In this example, we are specifying that the value must be present in the URI for the request to succeed. If the URI does not contain a value for the specified variable, an exception will be thrown and the request will fail.


          
           
@GetMapping("/example/{exampleId}")
public void getExample(@PathVariable(required = true) int id) {
   System.out.println("The ID is: " + id);
}

Conclusion

The PathVariable annotation is a valuable tool in the Spring framework, allowing developers to access dynamic parts of the URI and bind them to method parameters. It has two optional attributes, name and required, which allow developers to customize how their code works. With the PathVariable annotation, developers can quickly and easily create more powerful and dynamic applications.
