Introduction to java.security.CodeSource

The 

          
           
java.security.CodeSource
class is used to store the information that identifies a location from which code is retrieved during the security verification process. This includes the location from which the code was loaded and any cryptographic signatures associated with the code. Examples of this include an applet or application loaded from a web server, or from a local file. Both the authentication code as well as the associated permission code are stored in this class.

Constructors of java.security.CodeSource

There are two constructors used to create a CodeSource object:


          
           
public CodeSource(URL url, CodeSigner[] signers)
public CodeSource(URL url, Certificate[] certs)

The first constructor takes as parameters the URL from which the code is loaded and an array of CodeSigner objects. Each CodeSigner object contains the public key and private key used to generate a certificate. The second constructor takes as parameters the URL from which the code is loaded and an array of Certificate objects. The Certificate objects contain the public key and private key used to generate a certificate.
