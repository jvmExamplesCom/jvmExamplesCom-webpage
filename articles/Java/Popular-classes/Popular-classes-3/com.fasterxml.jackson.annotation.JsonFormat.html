Introduction to com.fasterxml.jackson.annotation.JsonFormat

The JsonFormat annotation is used for defining date and time formats when serialization or deserialization JSON. This helps to ensure that dates and times are properly formatted so as to make them readable by other applications.

Using the JsonFormat Annotation

The JsonFormat annotation has several attributes which can be used to define the desired format of a date or time field. The attributes include the following:

  • pattern: The pattern to use when formatting the date or time (e.g. "yyyy-MM-dd'T'HH:mm:ss").
  • timezone: The time zone in which to interpret the date or time (e.g. "GMT").
  • locale: The locale to use when formatting the date or time (e.g. "en_US").
  • shape: The shape to use when formatting the date or time (e.g. "STRING").

The JsonFormat annotation can be applied to either a field or an accessor method. When applied to a field, it has the same effect as if it were applied to all accessor methods for the field. For example, the following code applies the pattern "yyyy-MM-dd'T'HH:mm:ss" to the date field:


          
           
@JsonFormat(pattern="yyyy-MM-dd'T'HH:mm:ss")
private Date date;

Similarly, the following code applies the same pattern to the getDate() method:


          
           
@JsonFormat(pattern="yyyy-MM-dd'T'HH:mm:ss")
public Date getDate() {
    return date;
}

The following code examples demonstrate how to use the JsonFormat annotation to format dates and times for JSON serialization and deserialization. In each example, the date is formatted to "yyyy-MM-dd'T'HH:mm:ss" with a time zone of GMT and a locale of en_US.
