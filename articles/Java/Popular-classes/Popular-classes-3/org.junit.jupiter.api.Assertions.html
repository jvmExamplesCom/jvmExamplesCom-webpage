Introduction to org.junit.jupiter.api.Assertions

JUnit 5 provides an API for asserting conditions in tests. The Assertions class provides static factory methods for creating and combining assertions, as well as a set of assertion methods that cover most typical use cases. This tutorial covers the basics of JUnit 5's Assertions API.

Creating Assertions

The Assertions class provides static factory methods for creating assertions. These methods return objects that can be passed to assertion methods such as assertEquals() or assertNotNull(). For example, the following code creates a SimpleAssertion object:


          
           
SimpleAssertion assertion = Assertions.simple("My assertion message");

Assertion Methods

The Assertions class also provides a set of assertion methods that can be used to verify the behavior of a code snippet. These methods are divided into several categories such as primitive assertions, comparison assertions, and exception assertions.

Primitive Assertions

The primitive assertions assertTrue(), assertFalse(), and assertNull() allow you to verify the values of primitive types. For example, the following code assert that the value of a boolean variable is true:


          
           
assertTrue(myBooleanVariable);

Comparison Assertions

assertEquals("foo", "foo");

Exception Assertions
