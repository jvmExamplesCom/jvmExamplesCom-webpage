Java: com.bumptech.glide.load.engine.DiskCacheStrategy

Overview

The DiskCacheStrategy provides a framework for how an image loader should cache and load image files on disk. It is used in the Glide library, which helps developers efficiently deal with images in their applications. By specifying a DiskCacheStrategy, the developer can define when, where, and how often their images are cached. This enables the application to reduce network activity and speed up user experience.

Usage

When using Glide, the DiskCacheStrategy is specified with the load() or into() methods. For example, the following code will use the DiskCacheStrategy to cache an image for 30 days: 

          
           
Glide
  .with(context)
  .load(url)
  .diskCacheStrategy(DiskCacheStrategy.CACHE_FOR_30_DAYS)
  .into(imageView);
The DiskCacheStrategy accepts several parameters, including: 
  • ALL
    - All image data will be stored on disk.
  • NONE
    - No image data will be stored on disk.
  • CACHE_FOR_N_HOURS
    - Image data will be stored on disk for a specified number of hours.
  • CACHE_FOR_N_DAYS
    - Image data will be stored on disk for a specified number of days.
  • CACHE_FOR_N_WEEKS
    - Image data will be stored on disk for a specified number of weeks.
  • CACHE_FOR_N_MONTHS
    - Image data will be stored on disk for a specified number of months.

The DiskCacheStrategy provides an easy and efficient way for developers to cache images on disk.
