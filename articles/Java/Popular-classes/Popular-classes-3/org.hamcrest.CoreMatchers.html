Introduction to Hamcrest CoreMatchers

Hamcrest CoreMatchers is a library of matcher objects (also known as constraints or predicates) allowing test authors to express various constraints in an easy, readable manner. This library is also part of the Hamcrest framework, which further adds to the powerful testing features available to Java developers.

CoreMatchers provides useful matchers for various primitive types, such as boolean, Integer, and String, as well as for custom objects and collections. In conjunction with the JUnit testing framework, CoreMatchers makes writing expressive unit tests more accessible.

Example Usage of CoreMatchers

We can illustrate the utility of CoreMatchers by reviewing an example test case. Consider a simple class called Person, which has two fields: firstName and lastName. We want to test that Person has the proper behavior when instantiated with a valid name.


          
           
public class Person {
    private final String firstName;
    private final String lastName;

    public Person(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }

    public String getFirstName() {
        return firstName;
    }

    public String getLastName() {
        return lastName;
    }
}

To start, we will create a test class called PersonTests. In JUnit 5, we would use the @Test annotation to define a test method. Within the test method, we can use the assertThat() statement and provide a matcher expression to represent the expected outcome of the test.


          
           
@Test
public void shouldReturnCorrectName() {
    Person person = new Person("John", "Doe");
    assertThat(person.getFirstName(), CoreMatchers.is("John"));
    assertThat(person.getLastName(), CoreMatchers.is("Doe"));
}

The first assert statement in the example tests that the Person’s first name is John. The is() method provided by CoreMatchers returns a matcher object that the assertThat() method uses to ensure the expected value matches the actual value. Similarly, the second assert statement tests that the Person’s last name is Doe.

We can also use CoreMatchers to test the types of objects using the instanceOf() method. For example, we can test that the firstName field of the Person object is a String:


          
           
@Test
public void shouldHaveCorrectType() {
    Person person = new Person("John", "Doe");
    assertThat(person.getFirstName(), CoreMatchers.instanceOf(String.class));
}

Conclusion

The Hamcrest CoreMatchers library makes it easy for Java developers to write expressive unit tests. Using the assertThat() method in conjunction with various pre-defined matchers, developers are able to quickly identify when objects are not behaving as expected.

We have demonstrated a few basic usage examples of CoreMatchers, but there are many more matchers available. Consult the official Hamcrest documentation for a full list.
