android.support.annotation.IdRes

android.support.annotation.IdRes is an annotation used to denote resource identifiers (such as @+id/my_id) that are safe to be used for referencing components in the Android framework.

Purpose

android.support.annotation.IdRes was introduced in API level 11 (Android 3.0 Honeycomb) as a way to prevent errors related to the use of non-existent resource identifiers. By using this annotation, developers can ensure that the resources referenced in their code will exist and be available at runtime.

Usage

The @IdRes annotation can be used to add type safety to any code which references resources in the Android framework: 

          
           
@IdRes int myResourceId = R.id.my_id;

//The View corresponding to the id is now available
View myView = findViewById(myResourceId);
This annotation can also be used to inform Android Studio that a particular identifier value is safe to use. This allows the static analysis tools to check for validity of the resource identifier and provide valuable feedback to the developer.
