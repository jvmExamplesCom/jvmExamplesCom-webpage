What is java.io.InterruptedIOException?

java.io.InterruptedIOException is a type of checked exception that is thrown when an I/O operation has been interrupted. This class extends IOException, which is a subclass of Exception, and indicates that an I/O operation has been interrupted.

Example

Let’s take an example of trying to read a file but the thread has been interrupted before being able to finish the operation. This can lead to an InterruptedIOException being thrown.


          
           
try {
    InputStream inputStream = new FileInputStream(file);
    int data = inputStream.read();
} catch (InterruptedIOException e) {
    // This exception will be thrown if the thread was interrupted
    // before the read operation could finish
    e.printStackTrace();
}

Why Should We Handle InterruptedIOException?

InterruptedIOException should be handled because it indicates that an I/O operation was interrupted. Depending on the situation, this may lead to data loss or other undesired outcomes. For example, if an application is writing a large file and the thread is interrupted while writing that file, then the data written up to the interrupt will be lost.

How to Handle InterruptedIOException?

A common way to handle InterruptedIOException is to check if the thread was interrupted before attempting the I/O operation. If the thread is interrupted, then the I/O operation should be aborted. Typically, this is done with the Thread.currentThread().isInterrupted() method. For example:


          
           
if (Thread.currentThread().isInterrupted()) {
    // Abort the I/O operation
} else {
    // Attempt the I/O operation
}

The Thread.currentThread().interrupt() method can also be used to abort an I/O operation. This will set the thread’s interrupted status, which will cause an InterruptedIOException to be thrown on the next I/O operation.


          
           
Thread.currentThread().interrupt();
