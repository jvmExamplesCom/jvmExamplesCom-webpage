org.hibernate.SessionFactory

The SessionFactory is an interface in Hibernate which provides a static method to create a session object. It is used to get a current session from the database. The SessionFactory configuration is done using a standard XML configuration file. This file contains the database connection properties, mapping files for entities, and other miscellaneous settings.

Creating a SessionFactory

The SessionFactory is created by instantiating the org.hibernate.cfg.Configuration object. We can then call the buildSessionFactory() method on the Configuration object to create the SessionFactory instance. 

          
           
Configuration configuration = new Configuration();
SessionFactory factory = configuration.buildSessionFactory();
The Configuration object needs to be configured with the connection details, mapping documents, etc before the SessionFactory can be built.

Configuring the Connection Details

To configure the connection details, we can use the setProperties() method. It accepts a java.util.Properties object with connection parameters. The keys in the property object correspond to the values of the configuration parameters. 

          
           
Properties properties = new Properties();
properties.setProperty("hibernate.dialect", "org.hibernate.dialect.MySQLDialect");
properties.setProperty("hibernate.connection.driver_class", "com.mysql.jdbc.Driver");
properties.setProperty("hibernate.connection.username", "username");
properties.setProperty("hibernate.connection.password", "password");
properties.setProperty("hibernate.connection.url", "jdbc:mysql://localhost/dbname");

Configuration configuration = new Configuration(); 
configuration.setProperties(properties);

Mapping Entities

We can map our domain objects and persistent classes with the help of the addClass() method of the Configuration class. We can pass the class as an argument to this method. 

          
           
Configuration configuration = new Configuration();
configuration.addClass(User.class);
configuration.addClass(Address.class);
Alternatively, we can use an XML mapping file for mapping entities. For this, we can use the addResource() or the addFile() methods of the Configuration class to load the XML document containing the entity mappings. 

          
           
Configuration configuration = new Configuration(); 
configuration.addResource("user.hbm.xml");
configuration.addFile("address.hbm.xml");

Creating the SessionFactory

After configuration is complete, we can call the buildSessionFactory() method on the Configuration object to create the SessionFactory instance. 

          
           
Configuration configuration = new Configuration();
SessionFactory factory = configuration.buildSessionFactory();
We can use the same SessionFactory object to get multiple sessions in our application. We can also use the same SessionFactory instance across different threads in our application.

Getting a Session

Once the SessionFactory is created, we can open a new session using the openSession() method on the SessionFactory instance. 

          
           
Session session = factory.openSession();
This will return a session object associated with the database connection details. We must remember to close any open sessions when we are done with them by calling the close() method on the session instance. 

          
           
session.close();
