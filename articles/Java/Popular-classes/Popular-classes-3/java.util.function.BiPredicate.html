java.util.function.BiPredicate

A java.util.function.BiPredicate is a functional interface and one of the java 8's built-in functional interfaces. It is used when a two argument boolean method is needed as predicate. A BiPredicate returns a boolean and takes two arguments of any type.

Syntax


          
           
@FunctionalInterface
public interface BiPredicate<T, U> {
  boolean test(T t, U u);
}

Here Predicate is the generic type defined by java in the java.util.function package.

Example


          
           
//Suppose there is an arrayList of type Integer
ArrayList<Integer> numbers = new ArrayList<>();

//add some numbers
numbers.add(10); 
numbers.add(20); 
numbers.add(30); 
numbers.add(40);

//Now create a BiPredicate 
BiPredicate<Integer, Integer> b1 = (i1, i2) -> { 
   return i1 + i2 == 30; 
}; 

// test this BiPredicate 
for (Integer i : numbers) { 
   if (b1.test(i, 30 - i)) { 
      System.out.println("The pair is: " + i + ", " + (30 - i)); 
   } 
}

When we run the above code it produces the following result:


          
           
The pair is: 10, 20
The pair is: 20, 10
The pair is: 30, 0

Description

The BiPredicate interface can be used to check condition on two parameters and then return a boolean value. It is a functional interface with a single abstract method named test.

Methods of BiPredicate Interface

  • and(BiPredicate other): This method returns a composed predicate that represents a short-circuiting logical AND of this predicate and another. The expression will be evaluated using short-circuiting semantics.
  • negate(): This method returns a composed predicate that represents a logical negation of this predicate.
  • or(BiPredicate other): This method returns a composed predicate that represents a short-circuiting logical OR of this predicate and another. The expression will be evaluated using short-circuiting semantics.
  • test(Object t, Object u): This method is used to evaluate the given arguments according to the predicate.
