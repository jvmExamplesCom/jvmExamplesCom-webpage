Introduction to org.openjdk.jmh.annotations.Benchmark ​

The 

          
           
org.openjdk.jmh.annotations.Benchmark
annotation enables developers to create benchmark tests in Java. The annotation can be used for simple performance testing of a single operation, or to compare the performance of several operations.

Benchmark tests can be written in any Java environment, and JMH provides an easy way to build, run, and analyze results from a series of tests.

Creating a Simple Benchmark Test

Before we dive into some examples, let’s look at the simplest way to use the 

@Benchmark
annotation. All we need to do is the following: 

          
           
@Benchmark
public void someMethod() {
  // Some code here
}
This creates a benchmark test called 
someMethod
, which will measure the performance of the code inside it. Of course, this isn't very useful on its own, so we'll want to add some parameters when we're creating our tests.

Using Parameters with Benchmark Tests

We can use a number of different parameters when creating our benchmark tests. The 

@Param
annotation can be used to provide different input values to the method. For example: 

          
           
@Benchmark
@Param({"10", "100", "1000"})
public void testLoop(int max) {
  for (int i = 0; i < max; i++) {
    // Do something
  }
}

This benchmark test will be executed three times, with the parameter 

max
set to 10, 100, and 1000 respectively. We can also use the 
@State
annotation to provide a "state" object which will be initialized before each run of the benchmark test. This object can then be used inside the method. 

          
           
@State
public static class MyState {
  int max;

  @Setup
  public void setup() {
    max = 100;
  }
}

@Benchmark
public void testLoop(MyState state) {
  for (int i = 0; i < state.max; i++) {
    // Do something
  }
}
In this example, the 
MyState
class is initialized before each run of the benchmark test, and the 
max
field is set to 100. This value can then be accessed in the benchmark test method.

Using Results

When the benchmark test has been run, we can access the results in various ways. JMH provides a few methods for accessing the results, such as the 

getResults()
and 
getMaxScore()
methods. The 
getResults()
method returns a 

          
           
Map<String, Object>
, which contains the name of the test and the score. 

          
           
Map<String, Object> results = benchmark.getResults();
String testName = (String) results.get("TestName");
double score = (double) results.get("Score");
The 
getMaxScore()
method returns a 
double
value which is the score for the best result among all runs. 

          
           
double maxScore = benchmark.getMaxScore();

We can also access more detailed information about the results by using the 

getTimes()
method. This returns an array of 
double
values, which contain the time taken for each run of the benchmark test. 

          
           
double[] times = benchmark.getTimes();

Conclusion

This article has shown how to get started with the 

@Benchmark
annotation, including how to use parameters and how to access the results. With JMH, developers can quickly and easily create and run benchmark tests to measure the performance of their code.
