java.awt.GraphicsEnvironment Overview

The java.awt.GraphicsEnvironment class provides access to the local GraphicsEnvironment. It allows a Java program to find out which graphics devices are available, what their display capabilities are, and how to render to them via the java.awt.Graphics2D object. This class is used to determine the location of fonts on the local system, as well as platform-specific information such as the font rendering engine.

Using GraphicsEnvironment in Code

Getting an instance of GraphicsEnvironment is simple:


          
           
GraphicsEnvironment ge = GraphicsEnvironment.getLocalGraphicsEnvironment();

Once we have it, there are a variety of operations we can perform with it. The most common use-case is probably interacting with fonts:


          
           
// get all registered font families
String[] fontFamilies = ge.getAvailableFontFamilyNames();
// create a new Font instance using a registered font family
Font customFont = new Font(fontFamilies[0], Font.ITALIC, 12);

We can also get information about the current display device configuration:


          
           
// get the number of graphics devices available
int numDevices = ge.getNumScreens();
// get the size of the primary graphics device
Dimension primaryDeviceSize = ge.getMaximumWindowBounds().getSize();

Finally, we can create a new Graphics2D object by passing in a GraphicsConfiguration from our GraphicsEnvironment instance:


          
           
// get the default GraphicsConfiguration for the primary device
GraphicsConfiguration primaryConfig = ge.getDefaultScreenDevice().getDefaultConfiguration();
// create a new Graphics2D object with the primary graphics configuration
Graphics2D g2d = (Graphics2D)primaryConfig.createCompatibleGraphics();
