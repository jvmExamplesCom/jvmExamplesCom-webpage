javax.swing.event.DocumentListener in Java

Every application that interacts with a user needs some kind of input validation and responding to user inputs. The 


          
           
javax.swing.event.DocumentListener
interface permits the registration of a listener to observe changes in a text document model, providing notification whenever a change occurs.

Implementing DocumentListener in Java

In order to implement the DocumentListener interface, a class must implement all abstract methods defined in the DocumentListener interface. All implementations must be able to respond when a change occurs in the document. A typical implementation of a 

DocumentListener
may look like this: 

          
           
public class DocumentChangeListener implements DocumentListener {
 
    public void insertUpdate(DocumentEvent e)  {
        System.out.println("Document changed");
    }
 
    public void removeUpdate(DocumentEvent e)  {
        System.out.println("Document changed");
    }
 
    public void changedUpdate(DocumentEvent e)  {
        System.out.println("Document changed");
    }
}

Once the source code has been implemented, the DocumentListener can be registered to the TextField, JTextField, or JTextArea via the method 


          
           
addDocumentListener()
. The following code example shows how to register a 
DocumentListener
: 

          
           
JTextField textField = new JTextField();
textField.addDocumentListener(new DocumentChangeListener());

The DocumentListener will now receive notifications whenever a change is made to the text field.

Example of DocumentListener

JTextField textField = new JTextField();
textField.addDocumentListener(new DocumentListener() {

    public void insertUpdate(DocumentEvent e)  {
        String num = textField.getText();
        try {
            Integer.parseInt(num);
        } catch (NumberFormatException nfe){
            textField.setText("");
        }
    }
 
    public void removeUpdate(DocumentEvent e)  {

    }
 
    public void changedUpdate(DocumentEvent e)  {

    }
});
This code example adds a 
DocumentListener
to the text field. Whenever a character is inserted in the text field, the DocumentListener checks if it is a valid number or not. If the character is not a valid number, then it will reset the text field.

Conclusion

The 

          
           
javax.swing.event.DocumentListener
interface enables applications to respond to changes in the text document model. It provides an easy way to validate user inputs and react to changes in the text fields.
