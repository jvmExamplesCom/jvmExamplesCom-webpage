What is org.springframework.context.annotation.Scope in Java?

Scope is a key concept in Java to limit the visibility of objects, variables, methods and classes. It can be used to restrict the use of objects and variables within a given scope. The 


          
           
org.springframework.context.annotation.Scope
annotation defines the scope of a bean in the spring application context.

Overview of org.springframework.context.annotation.Scope

The 
@Scope
annotation is used to declare the scope of a bean when it is registered in the spring application context. The available scopes are: singleton, prototype, request, session, global session and application. The default scope is singleton. 

          
           
@Scope(value = "singleton") 
public class MyBean {}
The 
@Scope
annotation can also be used to control the visibility of a bean. For example, a bean can be declared as having a given ‘scope’ to indicate which other beans can access it. 

          
           
@Scope(value = "prototype") 
public class MyBean {}
When a bean is declared with the 
@Scope
annotation, it will be automatically registered in the spring application context.

Examples of org.springframework.context.annotation.Scope

Here are some examples of using the 
@Scope
annotation in various contexts.

Singleton Scope

In singleton scope, there is only one instance of a bean created in the entire application. All callers will get the same instance of the bean.


          
           
@Scope(value = "singleton") 
public class MyBean {}

Prototype Scope

In prototype scope, each caller will get a separate instance of the bean. This means that any state stored in the bean will not be shared among callers.


          
           
@Scope(value = "prototype") 
public class MyBean {}

Request Scope

In request scope, the bean is created for each HTTP request. This means that each request will get a separate instance of the bean.


          
           
@Scope(value = "request") 
public class MyBean {}

Session Scope

In session scope, the bean is created for each HTTP session. This means that each session will get a separate instance of the bean.


          
           
@Scope(value = "session") 
public class MyBean {}

Global Session Scope

In global session scope, the bean is created for each global HTTP session. This means that all sessions will get the same instance of the bean.


          
           
@Scope(value = "globalSession") 
public class MyBean {}

Application Scope

In application scope, the bean is created for each application context. This means that all application contexts will get the same instance of the bean.


          
           
@Scope(value = "application") 
public class MyBean {}

Conclusion

In conclusion, the 
@Scope
annotation is used to define the scope of a bean in the spring application context. It can be used to control the visibility of a bean, as well as to limit the usage of a bean’s instances. The available scopes are: singleton, prototype, request, session, global session and application.
