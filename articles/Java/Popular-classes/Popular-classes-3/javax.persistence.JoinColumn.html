javax.persistence.JoinColumn

Java Persistence API (JPA) is an object-relational mapping (ORM) framework for Java programming language which allows developers to access and manage relational databases from within their Java applications. One of the most commonly used elements of JPA is the JoinColumn annotation, which specifies the mapping between a source column and a target column in a given database table. The JoinColumn annotation provides various attributes that can be used for setting up a foreign key relationship between two different tables. These attributes include name, referencedColumnName, unique, nullable, insertable and updatable.

Example 1: Simple JoinColumn Usage

The following example shows how to use the JoinColumn annotation to set up a foreign key relationship between two different tables. In this example, the “Customers” table will be related to the “Orders” table through a foreign key relationship. 

          
           
@Entity
public class Customer {
    @Id
    private int id;
 
    @OneToMany(mappedBy="customer")
    List<order> orders;
 
    ...
}
 
@Entity
public class Order {
    @Id
    private int id;
 
    @ManyToOne
    @JoinColumn(name="customer_id")
    Customer customer;
 
   ...
}
</order>
In the above example, we have specified the foreign key relationship between the “Customers” table and the “Orders” table using the JoinColumn annotation. The JoinColumn annotation has been applied to the “Order” class and it specifies the following attributes:
  • name: This attribute specifies the name of the foreign key column in the “Orders” table.
  • referencedColumnName: This attribute specifies the name of the primary key column in the “Customers” table

Example 2: Unique and Nullable JoinColumn Attributes

The next example shows how to use the JoinColumn annotation to specify the unique and nullable attributes. The unique attribute specifies whether or not the foreign key column in the “Orders” table is unique. The nullable attribute specifies whether or not the foreign key column in the “Orders” table is allowed to contain a null value. 

          
           
@Entity
public class Customer {
    @Id
    private int id;
 
    @OneToMany(mappedBy="customer")
    List<order> orders;
 
    ...
}
 
@Entity
public class Order {
    @Id
    private int id;
 
    @ManyToOne
    @JoinColumn(name="customer_id", unique=true, nullable=false)
    Customer customer;
 
   ...
}
</order>
In the above example, we have specified that the “customer_id” foreign key column in the “Orders” table should be unique and not allowed to contain a null value.

Example 3: Insertable and Updatable JoinColumn Attributes

The last example shows how to use the JoinColumn annotation to specify the insertable and updatable attributes. The insertable attribute specifies whether or not the foreign key column in the “Orders” table can be inserted. The updatable attribute specifies whether or not the foreign key column in the “Orders” table can be updated.

In the above example, we have specified that the foreign key column in the “Orders” table cannot be inserted or updated.
