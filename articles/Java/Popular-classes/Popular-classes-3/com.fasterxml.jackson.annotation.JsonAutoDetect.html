What is JsonAutoDetect?

JsonAutoDetect is a Java annotation used with the Jackson JSON processor to indicate what properties of a Java object are to be serialized or deserialized. It allows developers to control the visibility of properties of a class that will be included in the output (serialized) JSON data.

How it Works

JsonAutoDetect works by specifying the property visibility for a specific type. It can be used on class level, field level and method level. The specified visibility determines how fields and methods will be exposed for JSON serialization and deserialization operations.

Here is an example of how JsonAutoDetect is used on a class:


          
           
@JsonAutoDetect
public class Person {
    private String name;
    private int age;

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

In this example, JsonAutoDetect is used to specify which properties should be exposed for serialization and deserialization. The @JsonAutoDetect annotation will expose all of the public getter and setter methods as well as any private fields declared in the class.

Visibility Levels

JsonAutoDetect provides several different levels of visibility for properties. These levels can be used to control which properties are exposed for serialization and deserialization. The available levels are:

  • Public: Public properties are always exposed for serialization and deserialization.
  • Protected: Protected properties are exposed for serialization and deserialization if they have public or protected accessor methods.
  • Default: Default properties are exposed for serialization and deserialization if they have public or protected accessor methods.
  • Private: Private properties are only exposed for serialization and deserialization if they have a public or protected accessor method.

Configuring JsonAutoDetect

JsonAutoDetect can be configured using the @JsonAutoDetect annotation. This annotation takes an argument of type JsonAutoDetect.Visibility. This argument indicates the visibility level that should be used when exposing fields and methods for serialization and deserialization.


          
           
@JsonAutoDetect(fieldVisibility = JsonAutoDetect.Visibility.PROTECTED)
public class Person {
    private String name;
    private int age;

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}

In the example above, the @JsonAutoDetect annotation has been used to configure the visibility level to PROTECTED. This means that only protected fields and methods will be exposed for serialization and deserialization operations.

Conclusion

JsonAutoDetect is a powerful annotation used to control the visibility of properties for serialization and deserialization operations. It can be used to control which properties should be exposed and which should remain private. Configuring JsonAutoDetect with the appropriate visibility level can help ensure that only the necessary properties are exposed.
