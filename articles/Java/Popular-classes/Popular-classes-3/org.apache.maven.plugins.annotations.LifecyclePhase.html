Annotations in

The 

          
           
org.apache.maven.plugins.annotations.LifecyclePhase
annotation is used in maven plugins to specify the build phase in which a mojo (plugin execution) must run. The annotation can be used in conjunction with the @Execute annotation to execute a mojo during the build cycle.

Syntax


          
           
@LifecyclePhase(phase)
The parameter 
phase
represents an enumeration type, defined as: 

          
           
VALIDATE,
INITIALIZE,
GENERATE_SOURCES,
PROCESS_SOURCES,
GENERATE_RESOURCES,
PROCESS_RESOURCES,
COMPILE,
PROCESS_CLASSES,
GENERATE_TEST_SOURCES,
PROCESS_TEST_SOURCES,
GENERATE_TEST_RESOURCES,
PROCESS_TEST_RESOURCES,
TEST_COMPILE,
PROCESS_TEST_CLASSES,
TEST,
PREPARE_PACKAGE,
PACKAGE,
PRE_INTEGRATION_TEST,
INTEGRATION_TEST,
POST_INTEGRATION_TEST,
VERIFY,
INSTALL,
DEPLOY,
PRE_CLEAN,
CLEAN,
POST_CLEAN,
SITE,
SITE_DEPLOY,

Example

The following example shows how to define a mojo with the @LifecyclePhase annotation:


          
           
@Mojo( name = "hello" )
@LifecyclePhase( LifecyclePhase.INSTALL )
public class HelloMojo extends AbstractMojo {

  // Mojo code

}
In this example the mojo 
HelloMojo
is configured to execute during the 
install
phase of the build cycle.
