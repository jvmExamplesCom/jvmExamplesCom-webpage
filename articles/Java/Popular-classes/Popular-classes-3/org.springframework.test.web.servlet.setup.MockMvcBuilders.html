Introduction to org.springframework.test.web.servlet.setup.MockMvcBuilders

MockMvcBuilders is a utility class provided by Spring MVC Test framework which provides a convenient way to initiate and configure MockMvc instance. It provides APIs to build MockMvc instances by registering one or more @Controllers or RequestMappingHandlerMapping, RequestMappingHandlerAdapter and ExceptionHandlerExceptionResolver instances.

Create a Standalone Setup using MockMvcBuilders

We can get started by creating a standalone setup with MockMvcBuilders. We can either use the static standAloneSetup() method or the MockMvcBuilders.standaloneSetup() constructor.


          
           
MockMvc mockMvc = MockMvcBuilders.standaloneSetup(controller).build();

The static standAloneSetup() method has two variants - one takes var-args of controller and other takes an Iterable of controllers. The same goes for the constructor.

Create a WebApplicationContext Setup using MockMvcBuilders

We can create a web application context setup with MockMvcBuilders. To do that, we need to first create an instance of WebApplicationContext and then pass it to the MockMvcBuilders.webAppContextSetup(context) method.


          
           
WebApplicationContext applicationContext = ...;
MockMvc mockMvc = MockMvcBuilders.webAppContextSetup(applicationContext).build();

Generate MockMvc Instance with Annotations

We can create a standalone setup using annotations. We can register our controller using the MockMvcBuilders.annotationDriven() method which will scan the controller classes for annotations like @Controller, @RequestMapping, @ExceptionHandler etc and use them to generate the MockMvc instance.


          
           
MockMvc mockMvc = MockMvcBuilders.annotationDriven().build();

We can also provide a custom instance of AnnotationConfigApplicationContext which contains all the configuration and definitions of beans which we want to inject in our controller.


          
           
AnnotationConfigApplicationContext context = ...;
MockMvc mockMvc = MockMvcBuilders.annotationDriven(context).build();

Configure Additional Components

We can also configure additional components in our MockMvc instance. For example, if we want to add custom filters or specify a custom WebApplicationContext then we can use the MockMvcBuilder.


          
           
MockMvc mvc = MockMvcBuilders.webAppContextSetup(context)
    .addFilter(filter)
    .addFilters(filters)
    .setViewResolvers(viewResolver)
    .setControllerAdvice(controllerAdvice)
    .build();

This is just a summary of some of the features provided by MockMvcBuilders. For more information, please refer to the official documentation.
