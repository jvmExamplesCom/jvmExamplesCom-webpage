Introduction to org.eclipse.jetty.server.handler.AbstractHandler

org.eclipse.jetty.server.handler.AbstractHandler is an abstract class of the Jetty server that provides a framework for a request handler that processes HTTP requests from clients. A request handler is any class that implements the Handler interface or extends the AbstractHandler class.

It helps in coordinating the processing of an incoming HTTP request through the declared sequence of subsequent Handlers, as well as managing the outgoing response.

Implementing AbstractHandler

The AbstractHandler class provides implementations of the main Handler interface methods, thus allowing it to be used as a base class for any custom Handler. It defines the following methods:


          
           
public ContextHandler getContext() 
public void handle(String target, Request baseRequest, HttpServletRequest request, 
                   HttpServletResponse response)
public void setContext(ContextHandler context)

The handle() method is the main one, and defines how the request will be processed. The other two methods are used for setting and retrieving the associated ContextHandler.

Example Usage

The following example shows a simple implementation of the AbstractHandler:


          
           
public class MyHandler extends AbstractHandler {
    public void handle(String target, Request baseRequest, 
                       HttpServletRequest request, HttpServletResponse response) 
        throws IOException, ServletException {
        response.setContentType("text/html;charset=utf-8");
        response.setStatus(HttpServletResponse.SC_OK);
        baseRequest.setHandled(true);
        response.getWriter().println("<h1>Hello World</h1>");
    }
}

The handle() method simply sets the content type and status code of the response, and then prints the "Hello World" message to the output stream.

Conclusion

In conclusion, org.eclipse.jetty.server.handler.AbstractHandler provides a convenient way to create custom handlers for HTTP requests. It makes it easy to coordinate the processing of requests by providing implementations for the main Handler methods.
