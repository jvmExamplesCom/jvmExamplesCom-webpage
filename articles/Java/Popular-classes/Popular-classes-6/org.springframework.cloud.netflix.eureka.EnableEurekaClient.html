org.springframework.cloud.netflix.eureka.EnableEurekaClient

The org.springframework.cloud.netflix.eureka.EnableEurekaClient annotation enables an application to become a Eureka client and be able to discover services. Once the annotation is enabled, the application registers itself with the Give Eureka server (or service) and will receive instances of services that have been registered.

This annotation has the following properties: 
  • @EnableEurekaClient
    – Enables the application to become a Eureka Client
  • 
            
             
@EnableDiscoveryClient
    – Enables the application to discover other services in your application

Example

Let's look at a simple example of how to use the @EnableEurekaClient annotation in a Spring Boot application.


          
           
@SpringBootApplication
@EnableEurekaClient
public class MyApp {

    public static void main(String[] args) {
        SpringApplication.run(MyApp.class, args);
    }

}

In this example, we enable the application to become a Eureka client. The application will then be able to discover other services registered in the Eureka server. Once the application is running, you can access the Eureka dashboard and you'll see that the application is registered in the Eureka server:


          
           
http://localhost:8761/

You can also access the application's health check endpoint:


          
           
http://localhost:8080/actuator/health

Conclusion

The 
@EnableEurekaClient
annotation enables an application to become a Eureka client and allow it to discover other services in the application. This can be very useful in large distributed applications where service-discovery of dependent services is needed.
