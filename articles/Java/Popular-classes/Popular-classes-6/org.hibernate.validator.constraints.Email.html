org.hibernate.validator.constraints.Email

The org.hibernate.validator.constraint.Email annotation is used to denote that a field is a valid email address. This annotation works on all Java Bean properties and method parameters. It will validate if the provided email is a well-formed address.

Defining Email Validation Annotation

The Email annotation can be added to fields like this:


          
           
public class User {

    @Email
    private String email;
}

Validation Rules

The Email annotation will validate a field against the following criteria:

  • The email must be of the format local-part@hostname.
  • The local-part (the part of the email before the @) must contain only alphanumeric characters, underscores (_), hyphens (-), or periods (.).
  • The local-part must be at least one character long.
  • The hostname must contain only alphanumeric characters, hyphens (-), or periods (.).
  • The hostname must be at least one character long.
  • The email must be a valid email address (for example 
    test@example.com
    ).

Configuring Email Validation Annotation

The Email annotation can be configured with a few parameters. These parameters are listed below:

  • message - The error message to be displayed when the validation fails.
  • groups - The validation groups to which this constraint belongs.
  • payload - Additional information about the validation.

Example

In this example, we show how to use the Email annotation. We define an entity which contains an email field. We then annotate this field with the Email annotation.


          
           
public class User {

    @Email(message = "Invalid email address", 
           groups = {Default.class}, 
           payload = Severity.Error.class)
    private String email;
    //getter and setter methods
}

Now, when a user tries to set an invalid email address on this field, the validation will fail and an error message will be displayed.
