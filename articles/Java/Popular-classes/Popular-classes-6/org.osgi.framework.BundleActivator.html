Introduction to org.osgi.framework.BundleActivator

The 

          
           
org.osgi.framework.BundleActivator
is a class used in the OSGi framework to manage Java bundles. The purpose of the BundleActivator is to provide an interface for a bundle's activation and deactivation within the OSGi framework. It includes two methods that are called when the bundle is activated or deactivated, allowing the bundle to execute code when these events occur.

Bundles are activated when they are installed and deactivated when they are removed or the OSGi framework is shut down. Activation and deactivation may also be triggered manually, or programmatically by another bundle. The BundleActivator provides a simple way of executing custom code at these times.

Using BundleActivator

To use the BundleActivator, you must create a class which implements the 


          
           
org.osgi.framework.BundleActivator
interface. This class contains two methods, 
start
and 
stop
, which are executed when the bundle is activated and deactivated respectively. 

          
           
public class MyActivator implements BundleActivator {

    public void start(BundleContext context) throws Exception {
        // execute code when bundle is started
    }

    public void stop(BundleContext context) throws Exception {
        // execute code when bundle is stopped
    }
}

When creating a BundleActivator, you should ensure that all the code within these methods executes quickly, since they are blocking the OSGi framework while they run. Once the methods return, the activation/deactivation of the bundle will be complete.

The BundleContext

The BundleContext is a parameter passed to the BundleActivator's 

start
and 
stop
methods. It provides the bundle with access to the OSGi framework's service registry, allowing the bundle to register and unregister services, as well as look up existing services.

The BundleContext can also be used to install additional bundles, retrieve information about the bundle hosting the BundleActivator, or listen for events like service registration and unregistration.

Example Usage

Let's take a look at a simple example to see how you can use a BundleActivator. In this example, we will create a BundleActivator which registers a basic service when the bundle is activated. We will then use the BundleContext to look up the registered service and display its details.


          
           
public class MyActivator implements BundleActivator {

    ServiceRegistration<String> serviceReg;

    public void start(BundleContext context) throws Exception {
        // register a basic service
        Hashtable<String, String> props = new Hashtable<>();
        props.put("service.name", "basic-service");
        serviceReg = context.registerService(String.class, "Hello World!", props);
    }

    public void stop(BundleContext context) throws Exception {
        // unregister the service
        serviceReg.unregister();
    }
}

Now let's use the BundleContext to retrieve the service we just registered. We can use the 


          
           
getServiceReferences()
method to search for services with the specified property. We can then iterate over the returned list to get the details of each service. 

          
           
// search for services with the name "basic-service"
Collection<ServiceReference<String>> refs = context.getServiceReferences(String.class, "(service.name=basic-service)");

// iterate over the returned references
for (ServiceReference<String> ref : refs) {
    System.out.println("Found service: " + ref.getProperty("service.name"));
    System.out.println("Value: " + context.getService(ref));
}

This example shows how you can use a BundleActivator to register and unregister services, as well as use the BundleContext to retrieve details about those services. You can use this same technique to perform any tasks you need at activation and deactivation time.

Conclusion

The org.osgi.framework.BundleActivator is a powerful tool for managing bundles within the OSGi framework. It provides an easy way to execute custom code at activation and deactivation time, using the BundleContext to interact with the framework's service registry. With this powerful API, you can easily add custom functionality to your bundles, allowing them to interact with other bundles and the OSGi framework itself.
