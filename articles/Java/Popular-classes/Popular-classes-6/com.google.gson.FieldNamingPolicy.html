Java - com.google.gson.FieldNamingPolicy

Java is a powerful, object-oriented programming language with an easy-to-learn syntax. One of the features of Java is its support for the com.google.gson.FieldNamingPolicy class, which allows developers to control how fields in their Java objects are named or mapped to JSON elements.

The FieldNamingPolicy class contains a set of predefined policies that can be used to determine how fields are named when converting from Java objects to JSON elements. The policies are as follows:

  • IDENTITY - Fields are mapped directly to JSON fields using the same name.
  • UPPER_CAMEL_CASE - Fields are mapped to JSON fields using the upper camel case naming convention (e.g. "MyField" becomes "myField").
  • LOWER_CAMEL_CASE - Fields are mapped to JSON fields using the lower camel case naming convention (e.g. "myField" becomes "myfield").
  • LOWER_CASE_WITH_UNDERSCORES - Fields are mapped to JSON fields using the lower case with underscores naming convention (e.g. "my_field" becomes "my_field").

To use the FieldNamingPolicy class, developers first create a GsonBuilder instance, then call the setFieldNamingPolicy() method on the GsonBuilder instance to specify the desired policy. The following code snippet shows how to use the LOWER_CAMEL_CASE policy:


          
           
GsonBuilder builder = new GsonBuilder();
builder.setFieldNamingPolicy(FieldNamingPolicy.LOWER_CAMEL_CASE);
Gson gson = builder.create();

Once the GsonBuilder instance has been created and configured, developers can use it to parse JSON strings into Java objects, or create JSON strings from Java objects.

Example

Let's look at a simple example to illustrate how the FieldNamingPolicy works. The following Java class contains two fields that represent an employee's first and last names:


          
           
public class Employee {
    private String firstName;
    private String lastName;

    public Employee(String firstName, String lastName) {
        this.firstName = firstName;
        this.lastName = lastName;
    }

    public String getFirstName() {
        return firstName;
    }

    public String getLastName() {
        return lastName;
    }
}

Given an instance of this Employee class, we can create a JSON string by using the Gson object created earlier with the LOWER_CAMEL_CASE policy. The resulting JSON string will have the fields "firstName" and "lastName" mapped to the corresponding Java fields:


          
           
Employee employee = new Employee("John", "Smith");
String json = gson.toJson(employee);
// json = {"firstName":"John","lastName":"Smith"}

// json = {"FirstName":"John","LastName":"Smith"}

The opposite operation, converting a JSON string to a Java object, is also supported by the FieldNamingPolicy class. In this case, the fields in the JSON string must match the names specified by the policy, or the parsing will fail.
