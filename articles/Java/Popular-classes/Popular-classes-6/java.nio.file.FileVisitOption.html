java.nio.file.FileVisitOption

java.nio.file.FileVisitOption is a type-safe enumeration that is used to specify how a file tree should be traversed. It provides two options: FOLLOW_LINKS and SKIP_SUBTREE.

FOLLOW_LINKS

The FOLLOW_LINKS option instructs a FileVisitor implementation to follow symbolic links in the given file tree. This means that any files or directories which are linked to in the file tree will also be visited and processed by the FileVisitor.


          
           
Path start = Paths.get("/path/to/start");
Files.walkFileTree(start, EnumSet.of(FileVisitOption.FOLLOW_LINKS), Integer.MAX_VALUE, new MyFileVisitor());

          

          
           
MyFileVisitor myFileVisitor = new MyFileVisitor();
Files.walkFileTree(start, EnumSet.of(FileVisitOption.FOLLOW_LINKS), Integer.MAX_VALUE, myFileVisitor);

SKIP_SUBTREE

The SKIP_SUBTREE option instructs a FileVisitor implementation not to traverse into sub-directories of the given file tree. This means that only the top-level files and directories of the given file tree will be visited and processed by the FileVisitor.


          
           
Path start = Paths.get("/path/to/start");
Files.walkFileTree(start, EnumSet.of(FileVisitOption.SKIP_SUBTREE), Integer.MAX_VALUE, new MyFileVisitor());

          

          
           
MyFileVisitor myFileVisitor = new MyFileVisitor();
Files.walkFileTree(start, EnumSet.of(FileVisitOption.SKIP_SUBTREE), Integer.MAX_VALUE, myFileVisitor);
