org.springframework.web.HttpRequestMethodNotSupportedException

The org.springframework.web.HttpRequestMethodNotSupportedException is an unchecked exception thrown when a request is made that doesn't match the supported methods of a specific endpoint. For example, if a method such as PUT or DELETE is used when only GET and POST are permitted.

The exception has the following constructors: 

          
           
public HttpRequestMethodNotSupportedException(String message)
public HttpRequestMethodNotSupportedException(String message, Throwable cause)
public HttpRequestMethodNotSupportedException(String[] supportedMethods)
public HttpRequestMethodNotSupportedException(String[] supportedMethods, String message)
public HttpRequestMethodNotSupportedException(String[] supportedMethods, String message, Throwable cause)
For example, if we make a REST call such as this: 

          
           
GET http://localhost:8080/hello

and the only requests allowed at that endpoint are POST requests, our application will throw an org.springframework.web.HttpRequestMethodNotSupportedException.

We can handle the exception in our code by using a try-catch block, like this:


          
           
try {
   // Make request
} catch (HttpRequestMethodNotSupportedException ex) {
   // Handle exception
}

We can also add a HandlerExceptionResolver to our Spring configuration to handle the exception globally and provide a custom error page:


          
           
@Configuration
public class MyWebConfig implements WebMvcConfigurer {
   
   @Override
   public void configureHandlerExceptionResolvers(List<HandlerExceptionResolver> resolvers) {
        SimpleMappingExceptionResolver simpleMappingExceptionResolver
                = new SimpleMappingExceptionResolver();
        Properties mappings = new Properties();
        mappings.setProperty(
                HttpRequestMethodNotSupportedException.class.getName(),
                "error/method_not_supported");
        simpleMappingExceptionResolver.setExceptionMappings(mappings);
        resolvers.add(simpleMappingExceptionResolver);
    }
}
And an example of a custom error page: 

          
           
<!DOCTYPE html>
<html lang="en">
<head>
   <title>404 Not Found</title>
</head>
<body>
   <h1>HTTP Request Method Not Supported</h1>
   <p>The requested HTTP method is not supported at this endpoint.</p>
</body>
</html>
