io.vertx.ext.web.Router

The io.vertx.ext.web.Router is a powerful tool for creating routes within a Vert.x web application. With the Router, you can create a range of different paths that can access anything from simple HTML pages to complex web services. It provides a simple and efficient way to handle routing, allowing you to define different types of route matchers and route handlers so that requests can be routed in the most appropriate manner.

Examples

Here are some examples of how to use the Router.

Example 1: Creating a Basic Route

The following is an example of creating a basic route that will serve a static HTML page. This route will match any request that contains the URL path “/page”.


          
           
Router router = Router.router(vertx);
router.route("/page").handler(routingContext -> {
    routingContext.response()
        .putHeader("content-type", "text/html")
        .end("<h1>Hello World!</h1>");
});

Example 2: Creating a Route with Parameters

The following is an example of defining a route that takes a parameter. The parameter we will be taking is an ID. This route will match any request that contains the URL path “/user/{id}”.


          
           
Router router = Router.router(vertx);
router.route("/user/:id").handler(routingContext -> {
    String userId = routingContext.request().getParam("id");
    // Do something with the userId
});

Example 3: Creating an Error Handler

The following is an example of defining an error handler for the router. This handler will catch any errors that occur in any routes defined within it and respond to those errors accordingly.


          
           
Router router = Router.router(vertx);
router.route("/").handler(routingContext -> {
    // Do something
});

router.errorHandler(500, routingContext -> {
    routingContext.response()
        .setStatusCode(500)
        .end("An error occurred!");
});

Example 4: Creating a Route with Regular Expressions

The following is an example of defining a route with a regular expression. This route will match any request that contains the URL path “/hello/{name}” and the name parameter is only allowed to contain alphanumeric characters. 

          
           
Router router = Router.router(vertx);
router.route("/hello/:name([a-zA-Z0-9]+)").handler(routingContext -> {
    String name = routingContext.request().getParam("name");
    // Do something with the name
});

Conclusion

As you can see, the io.vertx.ext.web.Router is a powerful tool that can be used to create routes within a Vert.x web application. You can create a range of different route matchers and route handlers to ensure that requests are routed in the most appropriate manner. If you need to do more complex routing, then the Router also provides a range of advanced features, such as regex matching, route grouping, and custom error handlers.
