java.time.DateTimeException

The java.time.DateTimeException class is an unchecked Exception that is thrown when a problem occurs during interactions with the java.time framework. This exception can be thrown from a variety of APIs that interact with the java.time framework, such as java.time.LocalDateTime, java.time.OffsetTime, and java.time.OffsetDateTime.

Examples

Here are some examples of how the DateTimeException may be thrown:


          
           
try{
    LocalDateTime dateTime = LocalDateTime.parse("2020-02-30 12:00");
} catch (DateTimeParseException e) {
    throw new DateTimeException("Invalid date/time format", e);
}

In this example, an attempt to parse an invalid date/time format is attempted, resulting in a DateTimeParseException. We then wrap this exception in a DateTimeException to provide more meaningful information to the caller.


          
           
try{
    OffsetTime offsetTime = OffsetTime.of(14,30,45,123, ZoneOffset.ofHours(-6));
} catch (DateTimeException e) {
    System.out.println("Invalid offset time " + e.getMessage());
}

In this example, an attempt to create an invalid OffsetTime is attempted, resulting in a DateTimeException thrown. The exception is then caught and the message is printed out.

Conclusion

The java.time.DateTimeException class provides a way for developers to handle errors related to the java.time framework. By catching this exception and providing meaningful messaging to the user, we can make our applications more robust and user-friendly.
