Java androidx.annotation.ColorRes

Java androidx.annotation.ColorRes is an annotation type in the AndroidX library to provide strongly-typed resources for colors. This helps to ensure that color references are consistently used and that only valid colors can be used as resources. It was introduced in Android Support Library version 23.2.0.

The androidx.annotation.ColorRes annotation should be used to refer to a color resource. To access the value of a color resource, use the ContextCompat's getColor() method with an int representing the ColorRes.

Examples

The following section provides an example of how to use the androidx.annotation.ColorRes annotation to refer to a color resource.


          
           
@ColorRes int myColor = R.color.colorPrimary;
int colorVal = ContextCompat.getColor(context, myColor);

Alternatively, if you just want to access the value of a color resource without using the androidx.annotation.ColorRes annotation, you can do this directly:


          
           
int colorVal = ContextCompat.getColor(context, R.color.colorPrimary);

The above code will refer to a color resource named “colorPrimary” which can be defined in your project's res/values/colors.xml file:


          
           
<resources>
  <color name="colorPrimary">#33b5e5</color>
</resources>

The androidx.annotation.ColorRes annotation can also be used when creating a ColorStateList resource. For example, the following code creates a ColorStateList object from a given ColorRes.


          
           
@ColorRes int colorRes = R.color.colorPrimary;
ColorStateList colorStateList = ColorStateList.valueOf(ContextCompat.getColor(context, colorRes));

Summary

In summary, the androidx.annotation.ColorRes annotation is an annotation type in the AndroidX library to provide strongly-typed resources for colors.
