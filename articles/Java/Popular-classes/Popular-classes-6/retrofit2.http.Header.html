Introduction to retrofit2.http.Header

Retrofit2.http.Header is an annotation used to indicate that the parameter is part of the request header.

It is used for api endpoints that require additional information about the request and provides a way to pass data to the server along with the request. For example, when making a request, you may need to provide additional data such as authentication credentials, language preferences, etc. This can be done using the Header annotation.

Usage of retrofit2.http.Header

Simple Example

The following is a simple example of how to use retrofit2.http.Header:


          
           
@GET("users/{userId}")
Call<User> getUser(@Header("authorization") String authToken,
                   @Path("userId") long userId);

In this example, the request header includes an authorization token that is required by the server. The token is passed to the server via the Header annotation.

More Complex Example

In some cases, multiple headers with various values need to be provided. In this case, the following snippet shows how to use the Header annotation to pass multiple headers:


          
           
@GET("users/{userId}")
Call<User> getUser(@HeaderMap Map<String, String> headers,
                   @Path("userId") long userId);

In this case, multiple headers with various values can be passed to the server in one go, as a Map of key-value pairs.

Conclusion

In conclusion, the retrofit2.http.Header annotation is a powerful tool for passing additional data to the API server. It is a convenient way to include authentication credentials, language preferences, and more.
