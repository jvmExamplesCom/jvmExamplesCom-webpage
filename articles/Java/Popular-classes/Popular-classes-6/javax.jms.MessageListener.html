javax.jms.MessageListener

Java Message Service (JMS) is an API used for messaging which provides a way for applications to post messages to other applications in a loosely coupled and asynchronous environment. JMS consists of two parts: the producer, which is responsible for creating the message, and the consumer, which is responsible for receiving and processing the message. The Java Message Service (JMS) API provides an interface to create a Message Listener which is responsible for receiving messages sent to it and then processing them.

Implementation

To implement the JMS Message Listener, we need to implement the 


          
           
javax.jms.MessageListener
interface. This interface is composed of a single method 
onMessage(Message)
, which is invoked when a message is sent to the listener. 

          
           
public class MyMessageListener implements MessageListener {
    public void onMessage(Message message) {
        // do something with the message
    }
}
Once the 
MessageListener
is created, it can be registered with the 
MessageConsumer
for receiving messages. The below snippet demonstrates how to register a 
MessageListener
with a 
MessageConsumer
: 

          
           
MessageConsumer consumer = session.createConsumer(destination);
consumer.setMessageListener(new MyMessageListener());
The 
MessageListener
will now receive all messages produced by the producer and sent to the destination, and it will call the 
onMessage()
method with the received message as argument. It is then the responsibility of the 
MessageListener
to process the message.

MessageListener
