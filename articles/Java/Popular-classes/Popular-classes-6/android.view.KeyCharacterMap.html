android.view.KeyCharacterMap

The android.view.KeyCharacterMap class is used to map characters to keys, allowing developers to create custom key bindings for user interface components. KeyCharacterMap provides methods to retrieve a mapping of a character to a key code and vice versa, query the type of hardware key code associated with a character, and determine whether a key is present on a device.

Android Key Character Map Overview

Android platform provides a set of predefined keyboard layouts that are available to all applications. This allows other applications to use these keyboard layouts in order to provide a consistent user experience across different apps. To meet this goal, Android provides a class called KeyCharacterMap. This class allows developers to define custom key bindings and map characters to their respective hardware key codes.

Creating Key Character Maps

The first step in creating a KeyCharacterMap is to create an instance of it. This can be done by calling the static factory method `KeyCharacterMap.load()` which takes in a `int` as an argument. The argument can be one of the following:
  • BUILT_IN_KEYBOARD: Used to create a key character map from a built-in keyboard.
  • VIRTUAL_KEYBOARD: Used to create a key character map from a virtual keyboard.
  • DEFAULT_KEYBOARD: Used to create a key character map from the default keyboard.
Once the KeyCharacterMap is created, you can use the `getKeyData()` method to get the mapping between characters and their corresponding hardware key codes. The `getKeyData()` method takes in a `char` argument and returns a `KeyData` object which contains information about the character and its associated key code.

Examples

Example 1: Defining a KeyCharacterMap

This example creates a KeyCharacterMap from the default keyboard: 

          
           
KeyCharacterMap kcm = KeyCharacterMap.load(KeyCharacterMap.DEFAULT_KEYBOARD);

Example 2: Getting Key Data

This example retrieves the key data for the character 'A': 

          
           
KeyData kd = kcm.getKeyData('A');
