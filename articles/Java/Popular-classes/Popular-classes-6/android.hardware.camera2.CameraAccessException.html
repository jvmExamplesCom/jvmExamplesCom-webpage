Understanding android.hardware.camera2.CameraAccessException

When developing applications for Android with the Android Camera 2 API, there are times when a CameraAccessException may be thrown. The CameraAccessException is an unchecked exception that indicates that a particular camera device could not be opened due to a device-specific error, a general camera error, or lack of camera permission. This article explores what CameraAccessException is, when it is thrown, and proposed solutions for resolving the issue.

What is android.hardware.camera2.CameraAccessException?

The CameraAccessException is a subclass of the RuntimeException and is thrown when a camera device could not be opened due to a device-specific error, a general camera error, or lack of camera permission. This exception indicates that the camera could not be opened for any number of reasons, including but not limited to:
  • Camera is already in use by another app
  • Camera's hardware or driver is out of date
  • Camera permission was not granted
  • An unknown error has occurred
The following code demonstrates how the CameraAccessException can be thrown when attempting to open a camera using the CameraManager class: 

          
           
CameraManager manager = (CameraManager) getSystemService(Context.CAMERA_SERVICE);
try {
    String cameraID = manager.getCameraIdList()[0];
    manager.openCamera(cameraID, cameraDeviceCallback, null);
} catch (CameraAccessException e){
    System.out.println("CameraAccessException Error: " + e.getMessage());
}

Reasons for CameraAccessException being thrown

The CameraAccessException may be thrown for a host of reasons, some of which are outlined below:

Camera Permission Error

If the app does not have permissions to access the camera, then the CameraAccessException will be thrown. In order to make sure that your app has the necessary permissions to use the camera, you should add the 

          
           
<uses-permission android:name="android.permission.CAMERA" />
line to your 

          
           
AndroidManifest.xml
file.

Camera Already in Use

If another application or service is currently using the camera, the CameraAccessException will be thrown when attempting to open the camera. In this case, you will need to wait until the camera is no longer in use before trying again.

Hardware or Driver Issues

If the camera's hardware or driver is out of date, then the CameraAccessException may be thrown as well. In this case, you should check for updates to the device's OS or check for any new drivers from the device manufacturer.
