Create DataOutputStream from FileOutputStream in Java

DataOutputStream is a class used to write primitive Java data types to an output stream, in particular for sending the data to a file. FileOutputStream is a class used to write bytes to a file or other resource location. As such, it is possible to create a DataOutputStream object from a FileOutputStream object in order to write primitive data types, such as ints, to a file.

Creating a DataOutputStream Object

The following example shows how to create a DataOutputStream object from a given FileOutputStream object: 

          
           
FileOutputStream fos = new FileOutputStream(path);
DataOutputStream dataStream = new DataOutputStream(fos);
Here, 

          
           
path
is a String object containing the location of the file which we are writing to. The 

          
           
fos
object represents a FileOutputStream object, and the 

          
           
dataStream
object represents a DataOutputStream object which we have allocated from the 

          
           
fos
object.

Writing Primitive Data Types to a File

Once we have our DataOutputStream object, we can use it to write primitive types to a file. For example, we can use the following code to write an int to our file: 

          
           
int value = 5;
dataStream.writeInt(value);
This will write the value 5 to the file represented by our 

          
           
dataStream
object. We can also write other primitive types such as longs and floats: 

          
           
long longValue = 10L;
float floatValue = 3.14;
dataStream.writeLong(longValue);
dataStream.writeFloat(floatValue);
The above code will write the long 10 and the float 3.14 to the file.

Closing the DataOutputStream Object

Finally, once we have finished writing data to our file, we need to close both the DataOutputStream object and the underlying FileOutputStream object. This can be done as follows: 

          
           
dataStream.close();
This closes the DataOutputStream object and its underlying FileOutputStream object. It is important to remember to close any streams that you have opened, as this will ensure that any data that has been written to them is flushed to the file. In addition, open streams can cause memory leaks if they are not closed.
