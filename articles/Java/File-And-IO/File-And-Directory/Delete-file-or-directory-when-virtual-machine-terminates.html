Delete Files & Directories When Virtual Machine Terminates

Java provides a number of methods for deleting files and directories from the file system. In this article, we'll take a look at how to delete a file or directory when the virtual machine terminates.

Deleting Files When VM Terminates

The 

          
           
java.io.File
class provides the static method 

          
           
deleteOnExit()
which deletes the specified file when the virtual machine (VM) terminates. This method is useful when you want to make sure that a file is deleted after your code has done with it. Here's an example: 

          
           
File file = new File("myFile.txt");
file.deleteOnExit();
In the above example, we create a 

          
           
File
object representing the file 

          
           
myFile.txt
and then call 

          
           
deleteOnExit()
on it. This tells the VM to delete the file 

          
           
myFile.txt
when it terminates.

Deleting Directories When VM Terminates

The 

          
           
java.io.File
class also provides an overloaded version of the 

          
           
deleteOnExit()
method which takes a directory as its parameter and deletes the entire directory and all its contents when the VM terminates. Here's an example: 

          
           
File dir = new File("myDir");
dir.deleteOnExit();
In the above example, we create a 

          
           
File
object representing the directory 

          
           
myDir
and then call 

          
           
deleteOnExit()
on it. This tells the VM to delete the directory 

          
           
myDir
and all its contents when it terminates.

Conclusion

In this article, we looked at how to delete a file or directory when the virtual machine terminates using the 

          
           
deleteOnExit()
method of the 

          
           
java.io.File
class. We saw how to use this method to delete files as well as entire directories and their contents.
